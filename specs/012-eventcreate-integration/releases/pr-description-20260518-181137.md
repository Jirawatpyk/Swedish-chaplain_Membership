## Summary

**F6 — EventCreate Integration** (`012-eventcreate-integration`).

The ninth feature of Chamber-OS, F6 replaces the manual Excel re-keying of event attendees with an automated, member-centric import. Two ingest modes ship in v1: a Zapier-mediated webhook for EventCreate tenants (15-min polling cadence) and a CSV upload path for tenants on Eventbrite / Meetup / Cvent / spreadsheet workflows (CSV is the **primary ingest mode** for non-EventCreate tenants, plus the backfill / outage recovery path for EventCreate tenants). Both modes pass through the same matching → quota → audit pipeline, with strict ACID-tx atomicity (FR-037) and admin-archive lifecycle (FR-019a). Ships **dark** behind `FEATURE_F6_EVENTCREATE=false`; 6 explicit pre-flag-flip operator gates remain on the ship-day checklist.

## Implementation

**Phases**: 10 phases × 162 tasks (156 closed in-session + 6 deferred-with-rationale to ship-day operator gates T150–T154a).

**Architecture highlights** (full provenance in retrospective + review history):

- **Two-mode ingest pipeline** sharing `processAttendeeInTx`: webhook (POST → HMAC verify → idempotency receipt → match → quota → audit) AND CSV import (batched 100-row tx with SAVEPOINT-per-row failure isolation). FR-027 webhook ↔ CSV equivalence by construction (not parallel impl drift).
- **HMAC-SHA256 signature verification** with timing-safe compare + 5-min replay-window protection + 7-day idempotency receipts + 24h secret-rotation grace window.
- **4-rule attendee → member matching**: (1) exact contact email, (2) email domain (with personal-email deny list), (3) fuzzy company-name (normalised + Levenshtein ≤ 3), (4) `non_member` fallback. Ambiguous fuzzy → `unmatched` for admin relink.
- **Strict ACID-tx atomicity** (FR-037): event upsert + registration insert + idempotency receipt + quota decrement commit in ONE tx; failure rolls back entire tx + emits `webhook_rolled_back` audit in a SEPARATE post-rollback tx (observability never lost).
- **Quota accounting**: partnership tier per-event ticket allotments + corporate tier annual cultural quotas. Refund credit-back. Over-quota persistence with informational badge (no in-product remediation v1).
- **Admin-archive lifecycle**: `archived_at` soft-delete + reversed `counted_against_*` flags + audit-logged credit-back per registration. No "unarchive" in v1.
- **Differentiated PII retention** per PDPA / GDPR data-minimisation: member-linked 5y, non-member 2y + pseudonymise. Admin-only erasure tool (`/admin/events/{eventId}/registrations/{registrationId}/erase`) satisfies PDPA §30 / GDPR Art. 17.
- **R2 atomicity invariant** (commit `f59782f8`): `maybeApplyStateChange` outer catch re-throws ALL `TxStageError` stages; debit-branch advisory lock added (mirrors credit-branch). Closes silent-swallow of `quota_decrement` failures that committed partial state.
- **R3 emitOrThrow boundary** (commit `7671fb94`): wraps `audit.emit()` raw throws (pool exhaust panic, sub-adapter regression) into `TxStageError('audit_emit')` at the helper. R3-T1 fault-injection seam exercises end-to-end on live Neon.
- **R4 27-caller safeAuditEmit migration** (commit `cb00ed1a`): every direct `audit.emit()` call site across `apply-quota-effect`, `archive-event`, `relink-registration`, `pseudonymise-stale-non-member-pii`, `erase-attendee-pii`, `toggle-event-category`, `create-event`, `force-expire-grace-secret`, `generate-error-csv-signed-url`, `generate-webhook-secret`, `rotate-webhook-secret`, `_helpers/emit-quota-scope-audit` migrated to `safeAuditEmit` helper. Raw throws now correctly classify as `audit_emit` in SRE alerts instead of leaking as `unknown`.
