Key strengths surfaced by this pass:

1. **Round-2 code-H1 "silent-success" fix is correct AND defended end-to-end.** The path-eventId guard now fires inside the use-case at step 1b — BEFORE any lock acquisition, audit emission, or DB mutation — and is wired through `runInTenantWithRollbackOnErr`, so a misrouted URL cannot leave partial state committed. Integration test 10 (`event_path_mismatch`) asserts both the error variant AND zero side effects. The route's surface-disclosure 404 + structured `admin_event_relink_eventid_path_mismatch` warn-log gives SRE a signal without leaking the discriminator to admins.

2. **Deadlock-safe sorted-key dual-lock acquisition (step 4b)** is the only correct way to support `A→B` and `B→A` concurrent relinks. Sort happens on the branded `LockKey[]` array which preserves the tenant + event + member composite — a sort on just `memberId` substrings would have been fragile. The same-member short-circuit at step 4 guarantees the two keys differ when both are computed. **Precedent matched**: archive-event uses `ORDER BY matched_member_id ASC` at the SQL layer for N rows; relink sorts 2 keys in-process. Both implement the same invariant.

3. **Double-layer pseudonymised guard** (Application step 2 pre-check + DB-layer `WHERE pii_pseudonymised_at IS NULL`) correctly handles the TOCTOU window between findById and updateMatchAndQuota. The repo discriminator probe at `drizzle-registrations-repository.ts:447–485` separates `invariant_violation` (row vanished — likely concurrent FR-032a erasure) from `pseudonymised_row_rejected` (concurrent retention sweep), and the use-case at step 7 promotes the latter back up to the top-level discriminator so the route's 409 mapping stays consistent with the step-2 pre-check.

4. **`adminOnlyWriterGuard` correctly closes E1 FR-035 drift.** The 6 admin write routes (relink + archive + 2 toggles + CSV import + manual event create) now uniformly return **403 to manager + 404 to member** (action-level vs surface-level distinction per spec.md:248-251), audit-emitting `role_violation_blocked` for both. Branded `UserId` return type eliminates re-wrapping at call sites. `getCurrentSession()` throw → typed 500 with `requestId`. Best-effort `tenantSlug` resolve logged at `debug` level so a sustained spike becomes observable.

5. **Constitution v1.4.0 Principle I sub-clause 3 (Review-Gate blocker) is GREEN.** The cross-tenant probe at `relink-registration.test.ts:1447–1556` mirrors `archive-event.test.ts` WARN-3 — Tenant A actor against Tenant B registrationId returns `registration_not_found` AND Tenant B's `matched_member_id` / `match_type` / `matched_contact_id` are byte-identical post-call. Defends against an inverted-RLS bug that might block SELECT but allow UPDATE.

6. **Test pyramid is well-balanced**: **29 unit+contract** (16 use-case mocked + 7 writer-guard + 6 members-search contract) + **11 integration on live Neon Singapore** + **3 E2E** (AS1 non_member→member + AS2 A→B credit-back + FR-014 pseudonymised disallowed). Brand-fixtures helper (`tests/helpers/brand-fixtures.ts`) introduces fixture-load-time UUID-v4 assertion so brand-bypass mistakes fail loudly in dev rather than silently propagating through tests.

7. **i18n coverage at 2833 keys × 3 locales GREEN** with 54 new relink-scoped entries (17 keys × 3 locales + columns.actions + auditEvents.registration_relinked). `errorToastNetwork` was added specifically for support-triage distinguishability between transport-layer and 5xx server failures.

8. **Comment-anchor citation pattern adopted**: stale line-number references (`:248`, `:46-52`) have been replaced with section anchors (`§ "POST handler init"`, `§ "Quota-side audit policy"`) plus real banner comments at the cited sites. This is exactly the convention that survives refactors — addresses Round-2 comments-M2 directly.
