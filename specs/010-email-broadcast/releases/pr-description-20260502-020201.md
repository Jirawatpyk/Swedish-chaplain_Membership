# F7 US4 — Public unsubscribe + tenant-scoped suppression

## Summary

Ships the recipient one-click unsubscribe surface for F7 Email Broadcast, fulfilling **PDPA §24 + GDPR Art. 21 + ePrivacy** obligations. Recipients click the unsubscribe link in any broadcast body (or one-click via RFC 8058 `List-Unsubscribe-Post` header) and land on a server-rendered `/unsubscribe/[token]` page that records the suppression in tenant-scoped `marketing_unsubscribes` and emits append-only audit events. The page is no-JS, locale-aware (EN/TH/SV), and contracted "never throws" — any infrastructure throw collapses to an i18n-localised "please try again" state, never a Next.js 500.

This PR closes Phase 6 / US4 (T136–T148) on the `010-email-broadcast` feature branch. **US5 (webhook tracking) was already shipped via #19; US6 + Phase 9 polish remain on this branch and will follow as separate PRs** matching the per-user-story shipping pattern of #18 (US3) + #19 (US5).

## Implementation

### New surfaces
- `src/app/unsubscribe/[token]/page.tsx` — server-rendered Node.js route. Pipeline: peek tid (no HMAC) → rate-limit → bind RLS → verify HMAC → use-case → render. 4 states: `success` / `already` / `invalid` / `error`
- `src/app/unsubscribe/[token]/error.tsx` — segment-level error boundary as last-line-of-defence safety net
- `src/modules/broadcasts/application/use-cases/unsubscribe-recipient.ts` — Application use-case: idempotent upsert + dual audit emit on first time only
- `src/modules/broadcasts/infrastructure/unsubscribe-token/hmac-signer.ts` — HMAC-SHA256 token signer + verifier + pre-tenant `peekTokenTenantId` resolver. Token format: `v1.<base64url-payload>.<base64url-mac>` carrying `{v, tid, bid, eml, lang?, iat}`. `crypto.timingSafeEqual` constant-time MAC compare; verify-before-parse to defeat JSON-DoS via forged tokens
- `src/modules/broadcasts/infrastructure/resend/email-template.ts` — `renderBroadcastHtml` with chamber-branded shell + locale-aware footer; `signUnsubscribeUrl` + `buildListUnsubscribeHeaders` for RFC 8058 + admin direct links
- `src/modules/tenants/domain/tenant-slug.ts` — **NEW** lightweight branded `TenantSlug` cross-cutting Domain module; `TenantContext.slug` is now `TenantSlug` (apples-to-apples comparison without coercion); deleted duplicate brand from `src/modules/plans/domain/plan.ts`

### Audit events (5y retention)
- `broadcast_unsubscribed` (first time only)
- `broadcast_suppression_applied` (first time only)
- `broadcast_unsubscribe_token_invalid` (every invalid-token path; NULL tenant on malformed)

### Telemetry
- `broadcasts.unsubscribes_total{tenant, outcome}` — bounded outcome enum (`success | already | invalid | rate_limited | repo_error | unhandled_error`)
- `broadcasts.unsubscribe_page_ttfb_ms{tenant}` — SLO-F7-006 p95 < 400ms
- `broadcasts.audit_emit_failed_total{event_type, tenant}` — Principle I signal-loss alert
- All metric calls wrapped in `safeMetric(...)` so OTel exporter throws cannot bypass the page's "never throws" contract

### Security hardening
- Anti-enumeration rate limit: 20 hits / 5 min per source IP (best-effort fail-open per GDPR Art. 21)
- Strict noindex/nofollow/noarchive metadata (signed token in URL must never be indexed)
- sha256 hashing of `(tenant_id+":"+email_lower)` and token plaintext in audit payload (no raw PII)
- Pre-tenant peek + RLS bind + HMAC verify-under-bound-tenant — narrowest possible RLS-bypass window
