## Implementation

### New bounded context: `src/modules/broadcasts/`
- 4 aggregates: `Broadcast`, `BroadcastDelivery`, `MarketingUnsubscribe`, `RecipientSegment`
- ~20 Application use-cases including `submit-broadcast`, `approve-broadcast`, `cancel-broadcast`, `dispatch-scheduled-broadcast`, `process-webhook-event`, `reconcile-stuck-sending`, `unsubscribe-recipient`, `cancel-in-flight-broadcasts-for-member` (F3 archival cascade)
- Infrastructure: Resend Broadcasts gateway, Svix HMAC-SHA256 webhook verifier, DOMPurify strict-allowlist sanitiser, HMAC unsubscribe-token signer/verifier, per-tick memoized members-bridge (R6 W-P3), F3 + F2 bridge adapters

### New API surfaces
- `/api/broadcasts/*` — member compose / submit / cancel / list (route helpers)
- `/api/admin/broadcasts/*` — admin queue / approve / reject / cancel / proxy-submit
- `/api/webhooks/resend-broadcasts` — Node-runtime webhook (Svix HMAC + 64KiB body cap + verify-before-parse)
- `/api/cron/broadcasts/dispatch-scheduled` — 5-min cron-job.org HTTP trigger (Bearer auth, FOR UPDATE SKIP LOCKED + advisory lock)
- `/api/cron/broadcasts/reconcile-stuck-sending` — 15-min reconcile (24h timeout per FR-028)
- `/api/cron/broadcasts/prune-expired-drafts` — daily draft-prune
- `/unsubscribe/[token]` — public, no JS, server-rendered, 4 states (success / already / invalid / error)

### Key architectural decisions
- **Tenant Isolation (Constitution I, NON-NEG)**: RLS+FORCE on all 4 F7 tables; `runInTenant(ctx, fn)` everywhere; webhook + unsubscribe pre-tenant bypass with peek→bind→verify pattern; 16-test cross-tenant integration suite
- **Quota timing (FR-003 / Q1)**: reserved at `submitted`, consumed at `sending → sent`
- **Cancellation cutoff (FR-004a / Q10)**: cancellable until `approved`; `broadcast_cancel_too_late` 409 once `sending`
- **HTML sanitiser (FR-002a / Q4)**: strict allowlist (no `<img>` per E9/X3 — tracking-pixel removed); 200KB body cap; 200char subject; URL schemes restricted to http/https/mailto
- **Recipient cap (FR-016a / Q7)**: 5,000 per single broadcast, enforced at submit AND dispatch
- **Cron concurrency**: per-(tenant, broadcast) `pg_advisory_xact_lock('broadcasts:'+tid+':'+bid)` + FOR UPDATE SKIP LOCKED; lock namespace disjoint from F4 `invoicing:` and F5 `payments:`
- **F6 EventAttendees stub-port (FR-015a / Q5)**: F7 ships stub returning `[]`; F6 swaps in real impl at F6 ship (Phase 2 batch release)
- **PCI**: N/A — F7 has no payment surface
