## Implementation

**New surfaces**:
- `POST /api/webhooks/resend-broadcasts` — Node runtime, Svix HMAC-SHA256 with ±5min tolerance, 64 KiB body cap, verify-before-parse invariant (Threat F-16 / OWASP A03), NULL-tenant audit on sig-reject, 410 Gone on kill-switch (Svix backoff terminal), 413 on body_too_large (distinct from 401 bad_signature for ops triage)
- `POST /api/cron/broadcasts/reconcile-stuck-sending` — 15-min cron-job.org cadence, FOR UPDATE SKIP LOCKED, MAX 50 rows/tick, 200 on `gateway_error` (per-row work already idempotent — natural retry next tick), 500 only on `uncaught_error`/`server_error` (genuine harness-retry semantics), dedicated `gateway_outage` log channel + per-tenant dedupeKey
- `src/modules/broadcasts/application/use-cases/process-webhook-event.ts` — root dispatcher with terminal-state guard, inserted-only `broadcast_concurrent_action_blocked` audit on late-event-after-terminal, 20-event noise floor on per-broadcast >5% complaint-rate auto-halt
- `src/modules/broadcasts/application/use-cases/reconcile-stuck-sending.ts` — 24h reconciliation with discriminated `RetrieveBroadcastOutcome` distinguishing `'present'` (transition + quota consume) from `'not_found'` (failed_to_dispatch, NO quota consume)
- `src/modules/broadcasts/infrastructure/resend/resend-broadcasts-webhook-verifier.ts` — Svix HMAC verifier with 5 discriminated `WebhookSignatureError` kinds + Object.hasOwn prototype-pollution defence + base64 round-trip secret-decode probe + empty-secret guard
- `src/modules/broadcasts/infrastructure/db/drizzle-broadcast-deliveries-repo.ts` — `onConflictDoNothing` on `(tenant_id, resend_event_id)` UNIQUE for FR-025 idempotency + explicit-switch aggregate buckets
