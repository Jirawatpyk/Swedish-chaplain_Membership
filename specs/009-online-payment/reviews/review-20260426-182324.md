| W1 | 🟡 Warning | `src/modules/payments/application/use-cases/sweep-stale-pending-refunds.ts` | 91–131 | **Postgres tx-abort semantics defeat per-row continue logic.** All rows sweep inside one `withTx`. When `updateStatus` for row N throws, the outer JS catch consumes it (`skippedCount += 1`), but Postgres marks the surrounding transaction as aborted. Subsequent statements in the same tx (rows N+1, N+2…) error with `current transaction is aborted, commands ignored until end of transaction block`. The "skip + continue" intent therefore degrades to: first-error halts effective progress; remaining rows all increment `skippedCount` but do nothing useful, and the outer `withTx` itself rolls back at commit time → swept rows revert. | Either (a) move the per-row try inside its own `tx.transaction(...)` SAVEPOINT so each row is independent, or (b) accept all-or-nothing and remove the per-row catch (let the outer try/err return `sweep_failed`). Option (a) preserves the current intent. Add a regression test: mock `updateStatus` to throw on row 1 of 3, then assert row 2 still committed via a fresh repo read. The current unit test (`per-row error`) passes only because the mock `tx` is a `Symbol` with no real Postgres behaviour. |
| W2 | 🟡 Warning | `src/modules/payments/application/use-cases/sweep-stale-pending-refunds.ts` | 92–131 | **Audit-emit failure leaves orphan failed row without audit event.** `updateStatus` (state change) runs first, then `audit.emit`. If the audit emit throws, the per-row catch silently increments `skippedCount` — but the row is already flipped to `failed` with no audit trail. Violates Constitution Principle I sub-clause #4 ("audit ALL state-changes") and Principle X (Reliability — no partial-state on failure). | Reverse the order so audit emits BEFORE updateStatus inside the same tx; or wrap both in a SAVEPOINT and rollback the row's update if audit emit throws. Add a unit test where `audit.emit` throws for one row and assert that row's status was NOT updated (currently no test covers this branch — the existing per-row-error test only throws from `updateStatus`). |
