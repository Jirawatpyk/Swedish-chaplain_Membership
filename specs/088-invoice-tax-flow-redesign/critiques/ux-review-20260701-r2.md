# UX Review — Round 2 (best-UX): 088 Invoice / Receipt Tax-Flow Redesign

**Date**: 2026-07-01 (UX round 2) · **Lenses**: chamber-os smart-features · enterprise forms/micro-interactions · mobile + deep WCAG 2.1 AA
**Basis**: spec.md (FR-001…026, SC-001…010, US1…8), both contracts, tasks.md, docs/ux-standards.md, docs/smart-chamber-features.md
**Verdict**: round-1 folds all PRESENT (strong); a deeper pass surfaced **7 HIGH + ~17 MED + 4 LOW** best-UX items, two of which are correctness regressions the UX lens caught.

## Closure of Round-1 (all present) — 4 weak spots to tighten
- **W1** SC-009 says the SV strategy for Thai tax terms "MUST be defined" but never picks one → **lock keep-Thai-plus-gloss** now (else SV translators blocked).
- **W2** FR-019 admin permanent-fail alert points at an "existing admin document/renewal alert channel" the spec never names → **name/confirm the surface** (or add it).
- **W3** FR-016 badge/label/"see RC" scoped to **list rows only** → also render on the **invoice detail-page header**.
- **W4** SC-010 is bound to **axe-core**, which does NOT test WCAG 1.4.10 Reflow / 1.4.4 Resize / 2.5.5-2.5.8 Target Size → the whole mobile cluster below is uncovered.
