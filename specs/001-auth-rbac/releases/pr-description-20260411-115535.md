## Constitution Compliance

All 10 Constitution v1.3.0 principles re-checked. **0 violations**, 3 documented
deviations in `plan.md` § Complexity Tracking:

1. **Hosting region is Singapore, not Thailand** — no major cloud provider has a
   Thailand region; Vercel `sin1` is the closest. PDPA Section 28 cross-border
   transfer rules cover the deviation.
2. **Application-layer unit tests rely primarily on integration tests against
   live Neon** — security-critical use cases get branch coverage end-to-end via
   real DB rather than via stubbed unit tests.
3. **Solo-dev review substitution for the ≥2-reviewers rule** (Constitution
   v1.3.0 NEW) — SweCham is a single-maintainer project. The 5-check
   substitute (multiple `/speckit.review` passes + `/speckit.staff-review` ×
   3 agents + 480/480 test coverage + DB-level defence-in-depth + post-
   remediation verification) was applied. The deviation is reversible.

The Constitution itself was bumped to **v1.3.0** as part of this PR (commit
`ba48b33`) to formally introduce the public barrel rule (Principle III) and
the solo-dev substitute clauses (Principle IX + Gate 9 + Governance).
