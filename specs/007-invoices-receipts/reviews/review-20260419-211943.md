## 2. Findings (deduplicated, ranked by severity)

### 🔴 Blockers — must fix before ship

| ID | File:Line | Description | Recommendation |
|----|-----------|-------------|----------------|
| **N1** | `src/modules/invoicing/application/use-cases/update-tenant-invoice-settings.ts:126,136` | **Audit emit outside transaction.** The use-case calls `deps.tenantSettingsRepo.upsert(...)` at L126 and then `deps.audit.emit(null, {...})` at L136 with no transaction boundary between them. If the audit write fails (network blip / Postgres deadlock / append-only trigger abort), the settings row has landed but the `tenant_invoice_settings_updated` event is lost. Violates Constitution v1.4.0 Principle I sub-clause 4 (*audit events emitted in the same transaction as the state change*) — NON-NEGOTIABLE. All other F4 mutation use-cases correctly use `deps.withTx(async (tx) => { ... })`; this one is the outlier. | Wrap both calls in `deps.withTx`, threading the `tx` handle into `upsert` and `audit.emit(tx, {...})`. Repository + audit port already accept a `tx` override. Add an integration test that forces `audit.emit` to throw and asserts the settings row is rolled back. |
| **N2** | `src/modules/plans/application/list-plans.ts:110,121` | **Float VAT arithmetic re-introduced after C3c consolidation.** `vatRateNumber = Number(tax.vatRateRaw)` + `Math.round(fee * (1 + vatRateNumber))`. Works by accident for 7% (binary-clean); for 8.5%, 10%, 13.5%, or other rates a future tenant might configure, IEEE-754 rounding produces off-by-1-satang totals on specific fee amounts. Identical defect class to R7-B6 — C3c nav'd the VAT source to `getTenantTaxPolicy` but kept the pre-existing float math from the old `fee_config` read path. Violates FR-002 + FR-005 (Thai-tax amounts are legal figures). | Replace with the existing domain helper. Pattern used in R7-B6 on the detail page: `const vat = calculateVat(Money.ofSatang(BigInt(fee)), VatRate.ofUnsafe(tax.vatRateRaw)); const total = Money.ofSatang(BigInt(fee)).add(vat).satang;`. Add a unit test with a non-7% rate (`"0.0850"`) pinning satang-exact totals. |

### 🟡 Warnings — strongly recommended before ship

| **N3** | `src/modules/invoicing/application/use-cases/upload-tenant-logo.ts:61-91` | **Declared-MIME trust + missing pixel-bomb guard.** Code picks the MIME whitelist from `input.declaredMime` (client `file.type`, fully spoofable) and then derives the sharp pipeline + upload `contentType` from the same declared value. sharp itself decodes via magic-byte sniff, but mismatched declared vs detected leads to a format-confused output URL.
