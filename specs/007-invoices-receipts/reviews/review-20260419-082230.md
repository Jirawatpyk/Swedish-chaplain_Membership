## 1. Executive Summary — Verdict

### ❌ CHANGES REQUIRED

The F4 transactional core is well-engineered (FR-003 atomicity, snapshot immutability, `@js-joda` fiscal-year, RLS+FORCE on all 5 tables, advisory-lock §87 no-gaps). Six prior rounds eliminated the major structural bugs.

However, the parallel review surfaces **6 unique Blockers** that prevent ship — most concentrated in the **PDF storage / access model** (public Blob URLs with no TTL + predictable keys) and **missing surfaces that the proxy kill-switch already references** (logo upload, tenant-invoice-settings CRUD, member portal routes, wrong cron path). The MVP-slice claim cannot be verified end-to-end because US3 (member portal) has no HTTP surface and US4 (settings) has no admin UI.

## 7. Things done well (kudos)

- **FR-003 transactional atomicity**: `withTx` + `TxAbort<E>` throw-carrier guarantees rollback of sequence on any downstream failure. Tests (a/b/d/e/f/g/h-replay) PASS on live Neon.
- **Snapshot immutability**: 2-layer defence — DB trigger + CHECK constraint + Domain enforcement.
- **Money correctness**: BIGINT satang everywhere; `Money` value object on `bigint`; `numeric(5,4)` for VAT rate; `@js-joda` `Asia/Bangkok` for fiscal-year boundaries.
- **RLS+FORCE on all 5 F4 tables** + `chamber_app` GRANTs + `runInTenant` + allocator's tenant-context assertion (non-prod).
- **Sequence overflow** guarded at Domain (`DocumentNumber.of()`) + use-case (throws to rollback).
- **Audit-in-transaction**: `audit.emit(tx, …)` shares the TX with the state change.
- **Receipt PDF separation**: migration 0024 split prevents invoice/receipt key collision.
- **i18n**: 117 F4 keys × 3 locales, 0 missing (`pnpm check:i18n` green); typed-phrase confirmation locale-aware (`toLocaleUpperCase(locale)`).
- **Layout containers (006 standard)**: list = `TableContainer`, detail = `DetailContainer`, new = `FormContainer`; `pnpm check:layout` green.
- **Reduced-motion**: `motion-reduce:duration-0` on alert-dialog overlay + content; WCAG 2.3.3 satisfied.
- **Skip-to-content + landmarks**: present on F4 layouts (FR-042).
