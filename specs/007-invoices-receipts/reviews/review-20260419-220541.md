## 4. Spot-checks / potential regressions

No regressions found. I specifically checked:

- **withTx adapter cast safety** (`drizzle-tenant-settings-repo.ts:101–106`). The `tx as TenantTx` cast is safe because the only caller path (`updateTenantInvoiceSettings`) obtains the handle from `tenantSettingsRepo.withTx`, which goes through `runInTenant` that returns a Drizzle `TenantTx`. External callers cannot pass an arbitrary value without a deliberate type assertion.
- **Zero-VAT edge case** in `list-plans.ts` — `'0.0000'.split('.')` → `['0', '0000']` → `BigInt('0000') === 0n` → gross = fee. ✅
- **Overflow protection** in `registration_fee_satang: z.string().regex(/^\d{1,19}$/)` — 19-digit cap is within `BigInt(…)` range (2^63 ≈ 9.22 × 10^18). THB 92 quadrillion satang is more than any realistic membership fee. ✅
- **Rate-limit key namespacing** — the settings key `f4:settings:` distinguishes from the logo key `f4:settings:logo:` and from invoice-issue `f4:issue:`. No collision. ✅
- **Logo regex vs PATCH route assertion** — regex allows any tenant slug (`[A-Za-z0-9_-]+`); the explicit `startsWith` post-parse narrows to the caller's slug. Both are needed: regex gates malformed shapes early, the startsWith asserts ownership. ✅
