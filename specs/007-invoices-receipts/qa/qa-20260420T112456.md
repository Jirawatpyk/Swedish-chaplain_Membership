## Fixture Seeding (What Changed During QA)

Initial QA run showed 9 fixture-gated E2E tests skipping because the E2E member had no seeded invoices. To close the gap, the following was shipped:

1. **`scripts/seed-e2e-portal-invoices.ts`** — idempotent seeder that:
   - Creates/reuses `e2e-member-empty@swecham.test` (role: member, same password as main E2E seed)
   - Upserts `E2E Alpha Co` member linked to `e2e-member@swecham.test` with 3 invoices (2 paid + 1 open) in the 900000-series sequence namespace (reserved for test fixtures, never collides with real seq allocator starting at 000001)
   - Upserts `E2E Echo Co` member linked to `e2e-member-empty@swecham.test` with 0 invoices
   - Handles the `contacts_tenant_email_uniq` partial-unique collision by re-linking pre-existing F3 contacts instead of INSERT
   - Document numbers conform to `DocumentNumber.parse` format `{prefix}-{YYYY}-{NNNNNN}`
2. **`.env.local` additions** — 4 new env flags (`E2E_MEMBER_HAS_INVOICES`, `E2E_MEMBER_EMPTY`, `E2E_MEMBER_EMAIL_EMPTY`, `E2E_MEMBER_PASSWORD_EMPTY`)
3. **`tests/e2e/portal-invoices.spec.ts`** — AS3 test parameterised to sign in with the empty-member credentials (same member cannot simultaneously have 3 and 0 invoices)

After these changes, all 21 E2E tests (7 per browser × 3 browsers) passed in a single 2.6-minute run.
