## Root-cause + fix log

See [pending-verification.md](../pending-verification.md) "2026-04-22 QA session" sections for the full 15-item bug-fix table (10 invoice-settings fixes + 5 CN fixes). Highlights:

1. **`signIn` helper regex** (invoice-settings) — `/^\/admin(\/|$)/` matched `/admin/sign-in` itself, `waitForURL` returned before POST completed. The 2026-04-21 session mis-attributed this as a cookie/CSRF race; root cause was the helper itself.
2. **T120 dual-bind probe in `route.ts`** — rejected every X-Tenant-header request with 403. Gated behind `!env.tenant.xHeaderEnabled` so throwaway-tenant E2E can mutate. Safe because the env validator refuses `xHeaderEnabled=true` in production.
3. **Seeder snapshot-shape bug (`seed-f4-e2e-admin-fixtures.ts`)** — `tenantSnap.address` / `memberSnap.company_name` didn't match domain types → `primary_contact_email` undefined → outbox.enqueue zod throw → **HTTP 500 on CN POST**. Real production-class bug; the seeder's invoice rendered OK (shapeThai tolerates undefined) but any downstream credit-note / resend / re-render path would crash.
4. **AS1 full-credit amount mismatch** — test filled 1,070 THB against a 10,700 THB invoice (comment claimed "100% of total" but seeder total is `1_070_000n` satang = 10,700 THB). Invoice correctly transitioned to `partially_credited`, test expected `credited`, mismatch.
