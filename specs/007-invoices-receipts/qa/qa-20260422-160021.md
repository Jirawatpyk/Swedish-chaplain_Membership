## Failures Detail

### TC-07 — `pdf-routes-cross-tenant-probe.test.ts` FK ordering

**Steps to reproduce**:
```bash
pnpm test:integration tests/integration/invoicing/pdf-routes-cross-tenant-probe.test.ts
```

**Expected**: 3/3 green on live Neon.
**Actual**: `beforeAll` fails with `PostgresError: insert or update on table "invoices" violates foreign key constraint "invoices_member_fk"`. Three `it` blocks are `skipped` because the suite setup threw.

**Diagnosis**: Test fixture inserts invoices before seeding the `members` row they reference. Schema has composite FK `(tenant_id, member_id)` enforced since migration 0019. The test was last touched on 2026-04-21 in `43c1cda` (pre-R17-fix); the regression is not caused by the R16/R17/R18 fix batch.

**Not a product defect**: the PDF-route cross-tenant probe logic itself is covered by:
- `tests/integration/invoicing/tenant-isolation.test.ts` (17/17 green)
- `tests/integration/invoicing/tenant-invoice-settings-probe.test.ts` (1/1 green)
- `tests/integration/invoicing/portal-ownership.test.ts` (green)
- `tests/integration/invoicing/retention-member-archive.test.ts` (green)

**Fix path** (post-QA follow-up): update the `beforeAll` seeder in `pdf-routes-cross-tenant-probe.test.ts` to insert members before invoices. ≤10 lines.

### TC-01 — Thai-RD auditor findings

**🔴 BLOCKER**: `src/modules/invoicing/application/use-cases/upload-tenant-logo.ts:22` — `import sharp from 'sharp'`. Application layer imports an infrastructure-level library. Violates Constitution Principle III (Clean Architecture) in spirit; `sharp` is an I/O binary re-encoder that belongs behind a port.

**Fix path**: extract `ImageReEncodePort` + `sharp-image-reencode-adapter.ts`. Mechanical refactor, ~80 LOC.

**🟡 HIGH**: `receipt_number_prefix` column missing from DB schema. `tenant-invoice-settings-repo.ts` declares `receiptNumberPrefix?: string | null` as optional, but no migration creates the column. `record-payment.ts:201` has `settings.receiptNumberPrefix ?? 'RE'` — always falls through to `'RE'` fallback for separate-mode tenants.

**Impact assessment**: SweCham uses `receipt_numbering_mode='combined'` per `plan.md § Post-critique R2 P4`. The `!combinedMode` branch is unreachable for the live tenant. **Zero production impact** at Phase 10 ship. Must be fixed before any tenant chooses separate-mode.
