## Failure Details

### TC-001 / TC-002 / TC-007: Browser round-trip deferred

**Status**: 🔵 Deferred — not a code failure; fixture gap
**Reason**: US1 AS1/AS2 and US2 AS1 require a seeded `(tenant, member,
plan, invoice-settings)` tuple on the live Neon DB. Invoice-settings
seeder (`seed-f4-invoice-settings.ts`) is idempotent and landed in
Phase 2; a combined end-to-end seeder for `(member + plan +
e2e-admin-sign-in)` is tracked as T061 prerequisite work for Phase 10.

**Repro steps (manual)**:
1. `pnpm dev` — dev server on :3100
2. Visit https://localhost:3100/admin → sign in as `e2e-admin@swecham.test`
3. Seed at least one corporate plan for SweCham 2026 via existing `pnpm db:seed-admin` path
4. Create a test member via `/admin/members/new`
5. Visit `/admin/invoices` → click "New invoice" → paste member UUID + plan id + year 2026
6. Verify:
   - Draft created + navigated to `/admin/invoices/{id}`
   - Click Preview → PDF downloads with watermark, NO row in `tenant_document_sequences`
   - Click Issue → type "ISSUE" → confirmation → seq `SC-2026-000001` assigned
   - Click Record payment → fill form → confirm → status=paid, receipt PDF downloads
   - `audit_log` contains `invoice_draft_created`, `invoice_issued`, `invoice_paid` rows
   - `notifications_outbox` has 2 rows (`invoice_issued` + `invoice_paid` event_type in context_data)

**Severity**: Medium — blocks CP-4.3 staging smoke but not CP-4.5 which
is satisfied by code availability.

### TC-008: Combined-mode receipt label hard-coded

**Status**: ⚠️ Partial
**Current**: `invoice-template.tsx` selects label from `input.kind`:
`'credit_note'` → "ใบลดหนี้ / Credit Note"; anything else (incl.
`receipt_combined`) → "ใบกำกับภาษี / Tax Invoice".
**Expected (US2 AS2)**: combined mode should render "ใบกำกับภาษี /
ใบเสร็จรับเงิน" (tax invoice + receipt combined label) when rendering a
paid invoice.
**Fix**: extend the `titleTh` / `titleEn` selection in
`invoice-template.tsx` to branch on `kind === 'receipt_combined'`.
Estimated ~4 lines. Tracked as T065 polish item (currently marked
[X] but label-switch logic is the polish cut).
**Severity**: Low — CP-4.4 achievable with template tweak; does not
block MVP ship gate for the "admin can replace Excel" value prop.

### TC-006: Drafts-tab UI toggle deferred

**Status**: ⚠️ Partial
**Current**: `/admin/invoices` list defaults to `includeDrafts: false`
(per use-case default) — drafts are hidden. A "Drafts" tab or filter
pill with count badge is NOT present in the current UI.
**Fix**: add `<Tabs>` or segmented-control to `page.tsx` driving a
client-side state + `includeDrafts=true` query param. Estimated ~30
lines. Tracked as a Phase 10 polish item under T057 polish cut.
**Severity**: Low — workaround: admins currently query
`/api/invoices?includeDrafts=true` directly; full UI is Phase 10.
