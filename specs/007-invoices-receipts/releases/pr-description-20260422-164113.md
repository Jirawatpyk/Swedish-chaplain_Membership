## Summary

F4 delivers Chamber-OS's **fourth business feature** and the **Phase 1 "finish line"** — the workflow that replaces the SweCham Excel workbook for billing. Admins can now draft → issue → collect manual payment → void → issue credit notes against Thai-tax-compliant membership invoices. All PDFs are bilingual (TH + EN), satisfy Thai Revenue Department requirements (§86/1, §86/4, §86/9, §86/10, §87, §87/3), carry sequential tax-document numbers with a §87 "no-gaps" structural guarantee, and live 10 years under legal-obligation basis (PDPA + GDPR Art. 6(1)(c)).

**7 user stories** (US1–US7), **43 functional requirements**, **11 success criteria**, **5 new DB tables**, **18 audit event types**, **1,123 i18n keys × 3 locales** (EN/TH/SV).

**Architecture** (Constitution v1.4.0 Principle III — Clean Architecture NON-NEGOTIABLE):

- Pure Domain: `Money`, `VatRate`, `DocumentNumber`, `FiscalYear`, `ProRatePolicy`, 3 aggregate roots (Invoice + InvoiceLine, CreditNote, TenantInvoiceSettings), 4 enforcement policies, 1 calculation pair (`calculate-vat` + `calculate-credit-note-vat`).
- 11 Application ports + 20 use cases. `@react-pdf/renderer` + `sharp` + `@vercel/blob` all hidden behind ports.
- R19 port extraction: `ImageReEncodePort` — `sharp` now confined to a single adapter for Principle III hygiene (fixed during QA pass).
- R20 polish: `ImageReEncodeResult` is a discriminated union so TS narrowing makes "stale empty buffer" structurally impossible.

**Security surfaces**:

- Two-layer tenant isolation (Principle I v1.4.0, Review-Gate blocker) — `runInTenant(ctx)` + RLS + FORCE + policy on all 5 F4 tables. Verified by `tenant-isolation.test.ts` (17/17) + 4 probe tests (`pdf-routes-cross-tenant-probe`, `portal-ownership`, `tenant-invoice-settings-probe`, `retention-member-archive`).
- Cross-tenant probes emit dedicated audit events on every deny path (404-opaque, never 403).
- Logger `REDACT_PATHS` covers `tax_id`, `member_*_snapshot`, `signed_url_token`, `pdf_binary`, `recipient_email` (depth-2), `payment_reference` (defence-in-depth). Single source of truth exported from `src/lib/logger.ts`.
- R17-02 void two-phase commit integrity: dispatcher sha256-verifies prefetched Blob bytes against enqueued `expected_pdf_sha256` before attaching; mismatch → permanently fail + dual audit emit.
- R18-01 integration test on live Neon exercises the integrity-mismatch path.
- Content-Disposition CRLF-injection defence + ESLint D2 rule.

**Thai-RD compliance**:

- Sequential §87 "no-gaps" allocator: `pg_advisory_xact_lock` + `SELECT … FOR UPDATE` on `tenant_document_sequences`. Property-test coverage on 50-writer concurrent load (~10 s wall-clock, <30 s budget).
- Bilingual PDF templates for invoice / receipt (separate + combined) / credit-note / VOID-stamped variants. Deterministic (`Math.random` + `Date` stubbed) content-addressable Blob storage. SC-003 byte-identical guarantee.
- FR-038 tax-ID snapshot immunity: receipts + credit notes render issue-time member identity, never live.
- Buddhist Era display-only on TH section; storage is Gregorian UTC throughout.

**Performance (all within SC budgets with headroom)**:

- PDF render p95 = **88 ms** (budget 800 ms).
- Invoice list p95 = **324 ms** @ 5 k × 2 tenants (budget 500 ms, SC-005).
