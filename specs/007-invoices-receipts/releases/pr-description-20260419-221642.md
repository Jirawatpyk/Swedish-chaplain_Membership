## Summary

Ships the **F4 Invoicing & Thai-Tax Receipts MVP slice** for Chamber-OS / SweCham:
admin issues a Thai-RD-compliant bilingual invoice → member pays → receipt
auto-generated → member downloads PDF from the portal. All money is stored as
`bigint` satang; every tax document carries a §87 gap-free sequential number
per `(tenant, document_type, fiscal_year)` secured by a Postgres advisory lock
+ transactional atomicity.

**Ship decision** (per `tasks.md` CP-4.7 go/no-go): ship the MVP slice to
SweCham now. US5 (void), US6 (credit notes), US7 (timeline integration), and
Phase 10 polish (auto-email dispatcher, overdue derivation, retrospective,
manual SR + cross-browser passes) ship as follow-up PRs against the same
branch family.

- `infrastructure/` — Drizzle repos, `@react-pdf/renderer` adapter (Sarabun TTF embedded), Vercel Blob adapter, `@js-joda` Asia/Bangkok fiscal-year boundary, Postgres advisory-lock sequence allocator, F4 audit adapter

**5 new DB tables + 9 migrations** (`0019–0027` + R9 `0029`):
- `invoices`, `invoice_lines`, `credit_notes`, `tenant_invoice_settings`, `tenant_document_sequences`
- RLS + FORCE on every tenant-scoped table; `chamber_app` GRANTs; `runInTenant` + `app.current_tenant` binding
- Immutability triggers on both `invoices` + `credit_notes` (R8-S6)
- `tenant_fee_config` DROPPED in R9 — `tenant_invoice_settings` is the single authoritative source for currency + VAT + registration fee + legal identity

**Admin surfaces** (`/admin/**`):
- `/admin/invoices` (list) + `/admin/invoices/new` (draft form) + `/admin/invoices/[id]` (detail + preview + issue + pay dialogs)
- `/admin/settings/invoicing` (US4 settings form, fieldset-card layout matching F3)
- Bootstrap empty-state with "Configure Invoicing" CTA when `tenant_invoice_settings` is missing

**Member surface** (`/portal/**`):
- `/portal/invoices` (list) + `/portal/invoices/[id]` (detail) + PDF download (byte-streamed, `Content-Disposition: attachment`)

**APIs** (under F4 kill-switch + admin-context or member-context guard):
- `POST /api/invoices` (create draft), `PATCH /api/invoices/[id]` (update draft), `DELETE /api/invoices/[id]` (delete draft)
- `POST /api/invoices/[id]/preview` (watermarked PDF, no seq consumed)
- `POST /api/invoices/[id]/issue` (FR-003 atomic: seq allocate → PDF render → Blob upload → audit → commit)
- `POST /api/invoices/[id]/pay` (record payment, receipt PDF generated)
- `GET /api/invoices/[id]/pdf` (byte-streamed with `attachment` disposition)
- `PATCH /api/tenant-invoice-settings` (admin-only, strict zod, rate-limited 30/min)
- `POST /api/tenant-invoice-settings/logo` (sharp EXIF-strip + MIME/dimension + pixel-bomb guard, rate-limited 15/min)
- `GET /api/portal/invoices/[id]/pdf` (member-context guard + RLS)

**Observability**:
- 16 F4 audit event types registered (`invoice_draft_created` … `auto_email_delivery_failed`)
- `invoice_cross_tenant_probe` + `credit_note_cross_tenant_probe` emitted on all cross-tenant paths (Constitution Principle I clause 4)
