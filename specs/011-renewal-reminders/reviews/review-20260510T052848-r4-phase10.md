## HIGH

### H1 — `tests/e2e/manager-readonly.spec.ts:155` — RBAC test always 404s; never verifies RBAC

POST target `/api/admin/renewals/[cycleId]/send-reminder` does NOT exist. Real path is `/send-reminder-now` (route present at `src/app/api/admin/renewals/[cycleId]/send-reminder-now/`). The `expect([403, 404]).toContain(resp.status())` accepts 404, so the test passes silently — never exercising RBAC.

**Fix**: change path to `/send-reminder-now` AND drop 404 from accept list (`expect(resp.status()).toBe(403)`).

### H2 — `tests/e2e/manager-readonly.spec.ts:66` — DropdownMenuItem is `role="menuitem"`, not `role="button"`

Pipeline-row "Send reminder" affordance is rendered via `DropdownMenuItem` (`pipeline-table.tsx:372-377`) → ARIA role is `menuitem`. The test's `getByRole('button', ...)` returns 0 matches every time, so the disabled-loop never runs.

**Fix**: open the row's actions menu first, then assert `getByRole('menuitem', { name: /send reminder/i })` is absent OR the parent action-menu trigger is disabled for managers.

### H3 — `tests/e2e/renewal-i18n.spec.ts:55-83` (and 94-101, 178-187) — TH/SV locale tests fail at sign-in

`setLocaleCookie(page, 'th')` is called BEFORE `signInAsAdmin(page)`. Sign-in form then renders in TH (`อีเมล`) or SV (`E-post`); `signInAsAdmin` helper does `getByLabel(/email/i)` — neither localized label matches.

**Fix**: sign in first, then set locale cookie + re-navigate.

## MEDIUM

### M1 — `tests/e2e/renewal-i18n.spec.ts:128` — BE-display fallback regex too loose

`/256[5-9]/` matches any 4-digit substring 2565-2569 (UUID fragments, error codes). Combined with `if (!hasBEYear) test.skip(...)`, false positives let the test silently skip instead of asserting the BE display rule.

**Fix**: keep only the prefixed regex `/พ\.ศ\.?\s?256[5-9]/` OR scope to `[data-testid="cycle-expires-at"]`.
