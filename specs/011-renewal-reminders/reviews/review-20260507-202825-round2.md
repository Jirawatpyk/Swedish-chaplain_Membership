## Important Issues (8 found)

### I-1. Doc/behaviour mismatch on audit-query fallback
**File**: `src/modules/renewals/application/use-cases/reconcile-pending-reactivations.ts:159-171`
**Source**: code-reviewer + silent-failure-hunter (C1 reclassified)

Inline comment says "fall back to legacy equality-fire policy" but the implementation returns `new Set()` which under `decideRemindersToFire` causes **all threshold-crossed rungs** to fire — i.e. on a day-28 cycle that previously emitted T-7, the fallback re-emits T-7 (duplicate audit row → duplicate dispatcher email).

**Fix options:**
- (a) **Doc-only**: rewrite comment to "fall back to fire-all-crossed-rungs" + add `metrics.renewalsReminderAuditQueryFailures` counter for SRE visibility.
- (b) **Behaviour change** (preferred long-term): treat audit-query failure as fail-closed (skip reminder rungs this run, timeout still processes) — doc the trade-off in the inline comment + bump `logger.warn` → `logger.error` with `errorId`.

Either approach unblocks; (a) is a 3-LOC fix, (b) is ~10 LOC + new metric.

### I-2. F4 callback `tx?: unknown` cast has zero runtime guard
**File**: `src/modules/renewals/infrastructure/renewals-deps.ts:303-306`
**Source**: silent-failure-hunter (C2) + type-design-analyzer (R2-T7)

```ts
const existingTx = txUnknown !== undefined
  ? (txUnknown as Parameters<typeof markCycleCompleteFromInvoicePaid>[2])
  : undefined;
```

If F4 ever passes the wrong shape (a refactor wraps `tx` in instrumentation, F5 onwards copies the contract incorrectly, etc.) F8 silently runs against the wrong handle. Failure mode: **cross-tenant scope corruption** (Constitution Principle I) or `TypeError: tx.execute is not a function` deep in Drizzle.

**Fix options:**
- (a) **Defense-in-depth**: add a runtime brand-check `if (!isTenantTx(txUnknown)) { fall back to runInTenant }` — cheap, ~5 LOC.
- (b) **Type-strengthen**: replace `tx?: unknown` in F4's onPaidCallback with a phantom-typed opaque handle in shared `lib/tx-handle.ts`. ~15 LOC + 1 cross-module touch.
- (c) **Generic over tx type**: `<TTx>(evt, tx?: TTx) => Promise<void>` — F4 threads concrete `TenantTx` through, F8 reads same brand. ~1 LOC + brand assertion in F8.

**Recommendation**: (a) for Phase 5 ship, (c) as F9 cross-cutting cleanup.

### I-3. Drizzle `reminderAuditQueryRepo` adapter has no integration test

**Fix**: `tests/integration/renewals/reminder-audit-query.test.ts` — seed 3 rows for tenant A + 3 for tenant B with overlapping `cycle_id`s; assert tenant A query returns exactly tenant A's rows. ~50 LOC.

### I-5. I3 tx-thread test name is misleading
**File**: `tests/unit/renewals/application/use-cases/mark-cycle-complete-from-invoice-paid.test.ts:188-209`
**Source**: pr-test-analyzer (I-3)

Test titled "runInTenant NOT called" but the assertion only checks sentinel-tx identity flowing through `findByInvoiceIdInTx`. Because `vi.mock('@/lib/db')` makes `runInTenant` a passthrough that runs `fn({})`, the sentinel-identity check would still pass even if the production code accidentally re-wrapped in `runInTenant`. The test does NOT prove its own title.

**Fix**: convert `vi.mock('@/lib/db')` to use `vi.fn()` so calls can be counted; assert `runInTenantSpy.calls === 0` for the existingTx-threaded path AND `runInTenantSpy.calls === 1` for the legacy backward-compat path. ~15 LOC.

### I-6. T138 reminder-emit failures swallowed silently
**File**: `src/modules/renewals/application/use-cases/reconcile-pending-reactivations.ts:265-300` (`emitReminderAudit`)
**Source**: silent-failure-hunter (I2)

Each rung emit wrapped in try/catch + `logger.warn` then **continues** without:
- Counter metric
- Error in cron output (the `cyclesProcessed` field returns success even when 100% of emits failed)
- Distinction transient-DB vs constraint-violation

A misconfigured RLS could silently drop every reminder for weeks; the cron's "I sent 47 reminders" log would be a lie.

**Fix**: add `remindersFailed` counter to output struct + bump success-counter only on emit-success + bump `metrics.renewalsReminderAuditEmitFailures`. ~10 LOC.
