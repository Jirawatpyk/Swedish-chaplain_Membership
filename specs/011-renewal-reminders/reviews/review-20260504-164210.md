## Executive Summary

Round 6 fixed Round 5's findings but introduced **5 new Blockers** that surfaced under deeper inspection:

1. **B-R6-1 (PROD-DEPLOY-BLOCKER)** — Migration 0100 uses `CREATE INDEX CONCURRENTLY` inside drizzle's transaction wrapper. Postgres raises `ERROR 25001: CREATE INDEX CONCURRENTLY cannot run inside a transaction block`. **Migration will crash on `pnpm db:migrate`**, blocking every subsequent deploy. Existing migrations 0021 + 0054 explicitly avoid CONCURRENTLY for this exact reason — the precedent was missed.
2. **B-R6-2 (PCI BYPASS)** — `isPanLikeReference` NFKD normalisation does NOT decompose Arabic-Indic / Devanagari / Thai script digits to ASCII. The non-ASCII strip then removes them entirely, producing an empty string that `\d{13,}` cannot match. **A 16-character `٤١١١١١١١١١١١١١١١` PAN substitute is silently accepted** and persisted to `audit_log.payload`. Comment claims this is handled — false.
3. **B-R6-3 (Spec drift, NEW)** — Round 6 ADDED `actor_user_id` to data-model.md payload columns for both `renewal_cycle_cancelled` + `renewal_cycle_completed_offline`. But `actor_user_id` is in `AuditContext`, NOT in the typed payload (audit-port.md L148 explicitly states this; F8AuditPayloadShapes confirms). data-model.md is now out-of-sync with audit-port.md AND the typed shape AND the actual emit code.
4. **B-R6-4 (Spec drift)** — Migration 0100 creates `renewal_cycles_lapsed_tier_idx` but data-model.md § 4 Indexes has no entry. Constitution Principle VIII requires perf-relevant indexes to be documented in data-model.md.
5. **B-R6-5 (Test coverage parity)** — W-R5-5 tx-arg assertion was added to `cancel-cycle.test.ts` but NOT to `mark-paid-offline.test.ts`. Both use-cases share the same TOCTOU pattern; B2 guard half-covered. A regression where mark-paid-offline reverts to non-tx `findById` would slip through CI.

Plus 1 Critical security finding (PCI redaction): `f4Reason` redaction has zero test coverage — Constitution Principle IV NON-NEGOTIABLE.
