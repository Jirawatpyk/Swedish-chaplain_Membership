## Round 11 Fixes Applied

### vi.mock infra failures (3 files)

User feedback: "(3 pre-existing vi.mock infra fails) ต้องแก้ด้วย"

3 unit test files were failing at module-load with `Cannot read properties of undefined (reading 'url')` from `upstash-rate-limiter.ts:55`. The files mock `@/lib/env` but their `vi.importActual('@/modules/renewals')` calls trigger transitive loads of `renewals-deps.ts → upstash-rate-limiter.ts` (and one file → `resend-client.ts`) which read `env.upstash.url|token` and `env.resend.apiKey` at module-init.

**Fix**: stub the additional env subpaths in the existing `vi.mock('@/lib/env', ...)` factory of each file. Result: 36/36 tests in the 3 files now pass.

| File | Tests | Stubs Added |
|------|-------|-------------|
| `tests/unit/api/cron/renewals/per-tenant.test.ts` | 11 | `upstash` |
| `tests/unit/api/admin/renewals/settings/schedules.test.ts` | 18 | `upstash` + `resend` |
| `tests/unit/api/cron/renewals/at-risk-per-tenant.test.ts` | 7 | `upstash` + `resend` |

### preConsumeGate test coverage (Round 9/10 closure)

`verify-renewal-link-token.ts` had 80.48% branch coverage after Round 9/10 added the `preConsumeGate` parameter — the new branches (`gate undefined` vs `gate defined`; `'allow'` vs `'block'`) had no test cases. Added 4 unit tests:

1. Gate returns `'allow'` → markConsumed runs → success
2. Gate returns `'block'` → markConsumed NOT called + emitTokenInvalid emits generic 'member_not_found_in_tenant'
3. Gate runs BEFORE `cycle.status === 'completed'` early-return (Round 10 ordering invariant)
4. Gate-block on cycle_already_completed path also blocks before status check

### requestId-null fallback test coverage (R11 branch closure)

3 audit emit sites (cycle-completed, self-service-initiated, renewal_token_invalid) each had an `?? null` fallback for `input.requestId` to coerce undefined/null → explicit `null` for the audit envelope. Existing tests all passed `requestId: 'req-1'` (truthy), leaving the right side of `??` uncovered. Added 3 unit tests:

1. Success path coerces `requestId: null → null` on `renewal_self_service_initiated` emit
2. Cycle-completed path coerces `requestId: null → null` on `renewal_token_clicked_on_completed_cycle` emit
3. Reject path coerces `requestId: null → null` on `renewal_token_invalid` emit

### `/* v8 ignore */` pragmas on defensive paths

3 catch blocks where `e instanceof Error ? e.message : String(e)` had an untestable `String(e)` fallback branch — applied single-line `/* v8 ignore next */` ABOVE the ternary expression line (multi-line block comment pragma `next N` did not register the ternary correctly with vitest@2.1.9 v8 provider). Plus `/* v8 ignore start ... v8 ignore stop */` block around the `default: { _exhaustive: never }` arm in `mapVerifyErrorToReason` to skip the default switch branch + its body. Pattern matches established convention in `compute-at-risk-score.ts`, `recompute-at-risk-scores-batch.ts`.

**Result**: scoped coverage proves 100% on all 4 dimensions (`Statements: 198/198, Branches: 44/44, Functions: 3/3, Lines: 198/198`).
