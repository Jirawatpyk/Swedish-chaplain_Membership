## Findings Detail

### Spec narrative drift (informational, not a regression)

**Issue**: `spec.md` US1 AS3 narrative still says non-attending rows "are reported in the result summary with a clear reason ('Skipped: Status=Cancelled') but do not consume quota or pollute member match counts." This was authored pre-Option B+ when non-Attending statuses were uniformly skipped. Per the FR-007 Option B+ revision (line 138-148), non-Attending statuses now PERSIST as `event_registrations` rows with the corresponding `payment_status` (pending/waitlisted/no_show) — they just don't count toward quota.

**Effective behaviour (verified GREEN by `csv-status-mirroring.test.ts`)**:
- Attending → registration row + `payment_status='paid'` + counts quota
- Pending → registration row + `payment_status='pending'` + NO quota
- Waitlisted → registration row + `payment_status='waitlisted'` + NO quota
- No Show → registration row + `payment_status='no_show'` + NO quota
- Cancelled → routed via FR-018 cancellation cascade → `payment_status='refunded'` + quota credit-back
- Unknown/typo → `rowsSkipped` with reason

**Recommendation**: Optional follow-up — update US1 AS3 narrative wording in `spec.md` to reflect the persisted-but-non-counting semantics. Code + tests are authoritative; narrative drift is cosmetic.
