### E2E Failure Analysis

All 21 E2E failures are the **same root cause**: `waitForSelector('[data-slot="table"]')` times
out because `page.goto('/admin/members')` redirects to `/sign-in` — there is no signed-in session.

The `tests/e2e/members-bulk-actions.spec.ts` `beforeEach` block comments "assumes E2E auth
fixture" but no `storageState`-based auth setup exists in `global-setup.ts` or the project
config. This is a **pre-existing infrastructure gap** that affects ALL F3 E2E specs, not a US4
regression.

**Recommendation**: Create a Playwright auth project (`setup` dependency) that signs in as
`e2e-admin` and saves `storageState` for downstream projects. This is an F3 infra task, not
US4-specific.
