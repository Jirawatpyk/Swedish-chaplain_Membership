### Key architectural decisions

1. **Throw-to-rollback pattern for atomic state + audit** — `UseCaseAbort<E>` sentinel thrown inside `runInTenant(ctx, async (tx) => {...})`, caught outside. Drizzle tx rolls back on throw only; a bare `return err(...)` would commit. All 14 tenant-scoped use cases use this pattern consistently.
2. **Audit ownership in Application layer** — Infrastructure repos are pure CRUD; use cases call `AuditPort.recordInTx(tx, ctx, event)` within the same tx so state + audit commit atomically (or roll back together).
3. **Cross-module boundary via adapter ports** — `InvitationCascadePort` (single allowed crossing point between F3 archive and F1 `invitations` table) preserves R001 column-grant semantics inside the adapter (`.returning({ userId })` only, never `{ id }`).
4. **Outbox-first notifications** — all F3 emails (+ F1 invitation post-T049) flow through `notifications_outbox` + cron dispatcher. Eliminates silent-success bugs and gives FR-012c retry semantics for free.
