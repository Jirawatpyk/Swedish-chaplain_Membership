- **Real bug #1 — FIXED 2026-05-28 (global): base-ui toggle fields have no accessible name** (`aria-toggle-field-name`, serious). Root (broader than first thought): both `src/components/ui/radio-group.tsx` (`Radio.Root`) **and** `src/components/ui/checkbox.tsx` (`Checkbox.Root`) are built on **base-ui**, which renders the `role="radio"`/`role="checkbox"` element with its own internal id and **ignores the passed `id`**; a sibling `<Label htmlFor="…">` therefore can't associate, leaving the control with no name. Surfaced on `/portal/broadcasts/new` — the actual failing node was the `schedule-immediate` **Checkbox** (`data-slot="checkbox"`), not a radio; the segment radios were a parallel instance. **Pre-existing (base-ui migration), NOT F9-caused.** **Fix applied:** per-consumer `aria-label` on every toggle field that relied on `<Label htmlFor>` — 5 RadioGroupItem consumers (`segment-picker`, `snooze-dialog`, `admin-preferred-locale-card`, `approve-dialog` ×2, `preferred-locale-form`) + 2 Checkbox consumers (`schedule-picker` `schedule-immediate`, `webhook-secret-reveal` `secret-saved-checkbox`). The `queue-table-client` (×2) and `members-table` (selectAll + selectRow) checkboxes already carried `aria-label`. **Verified:** `/portal/broadcasts/new` axe scan now passes on attempt #1 on a warm server (`qa/responses/f7-compose-checkbox.log`, chromium, 19.2s, no retry); typecheck clean.
- **Real bug #2 — F6: event-detail load path 500** (`PostgresError: column "status" does not exist`, reached via `runLoadEventDetail`). **STILL OPEN.** Confirmed NOT in `registrationsRepo.findByEventId` (it uses `paymentStatus`/`matchType`/`registeredAt`, no bare `status`) and there is no `text('status')` in the events/registrations Drizzle schema — so the offending reference is inside one of `eventsRepo.findById`'s sub-selects/joins and needs **live generated-SQL inspection** to pinpoint (not resolvable by code-reading). Affects the erase page and potentially `/admin/events/[id]`. **Pre-existing, NOT F9-caused.** Potential production 500.

**Recommendation:** bug #1 is **resolved app-wide** (every base-ui toggle field now has an accessible name). Open one dedicated ticket for (2) — the F6 event-detail `status` column 500 (priority — possible production impact; needs F6-domain live-query debugging). Neither blocks F9 Slice A (✅ after TC-019).
