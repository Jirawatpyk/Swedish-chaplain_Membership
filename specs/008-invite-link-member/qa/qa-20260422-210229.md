### The 2 failures — confirmed flaky, not regressions

- `tests/contract/invite-with-member.test.ts > (b) invalid memberId UUID format` — timed out at 10000ms in full-suite run

**Re-run targeted:** `pnpm test tests/contract/invite-with-member.test.ts tests/contract/members/member-invoices-route.test.ts` → **21/21 PASS in 10.21s**

Root cause: Vitest parallel worker contention on Windows when ≥150 test files run concurrently. Both suites have external-boundary setup (db + MSW + zod schema import) that exceeds the 10s per-test timeout under load. Identical tests pass in <3s when run without full-suite contention. Pattern matches user's recorded memory "E2E workers=1 mandatory" and is NOT caused by this branch.
