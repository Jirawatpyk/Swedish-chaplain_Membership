## Race Window Analysis (4-Layer Dispatch Defence)

**Verdict: Sufficient for current scale + deployment topology**

| Layer | Mechanism | Strength |
|---|---|---|
| 1 | cron-job.org 5-min tick spacing | Probabilistic |
| 2 | `FOR UPDATE SKIP LOCKED` on broadcasts row | Structural at broadcast level |
| 3 | Idempotency unique index on batch_manifests | Structural at INSERT |
| 4 | **Row-state guard** `updateStatus WHERE status=fromStatus + .returning()` | **Load-bearing at-most-once gateway-call guarantee** |

**One latent race documented as accepted**: Two cron ticks closely-spaced both pass Layer 2 (broadcasts row lock auto-released after eligible scan); both enter per-broadcast dispatch loop; Layer 4 row-state guard ensures only one actually calls Resend — the other logs `invalid_state_transition` and exits gracefully. Documented at `noop-advisory-lock.ts:24-30` + plus rationale for NOT wiring `pgAdvisoryLockAdapter` (pool exhaustion risk holding Postgres connection across multi-second Resend HTTP RTT).

## Top Strengths (collated from 3 reviews)

1. **Branded type discipline** — `IdempotencyKey` + `ChamberSubstitutedBody` + `Hostname` are nominal brands with single-entry-point constructors; XSS-via-tenant-name is compile-impossible.
2. **`emitCrossTenantProbe` exhaustive switch** — `assertNever(input.surface)` guard ensures adding a 4th probe surface produces a TypeScript compile error, not a silent audit gap.
3. **`safeAuditEmit` with adapter-invariant re-throw discrimination** — distinguishes transient storage failures (swallowed + metered) from programmer-bug invariants (re-raised as 5xx).
4. **DB-level XOR constraint (migration 0179)** on `broadcasts_template_provenance_xor` — converts future out-of-band corruption to loud Postgres 23514 error.
5. **Architecture barrel test** — shadows the ESLint barrel-guard rule (which has a known config bug) and freezes the 40-item deep-import backlog with file::importPath drift-resistant keys.
6. **Pipeline-order invariant** (`upload-inline-image.ts`): bytes-in-memory → ClamAV scan → IF clean THEN Blob upload — linear control flow with no branch skipping the scan.
7. **H10 atomicity test pattern** (`manage-image-allowlist.test.ts:238-299`): triple-pin assertion (use-case-survives + Result-shape-correct + metric-counter-incremented) — best-in-class fail-soft verification.
8. **Cross-tenant probe trilateral assertion** (T128): result.error.kind + audit row payload + target row untouched — catches both silenced-probe AND incorrectly-mutated-target regressions.
9. **`buildPerfBody()` Thai-script UTF-8 byte accounting** — correctly uses `Buffer.byteLength(filler, 'utf8')` to model 3-byte Thai codepoints; combined with warm-up call eliminates cold-pool bias.
