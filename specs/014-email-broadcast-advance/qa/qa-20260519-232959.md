## Failures Detail (resolved during this QA run)

### F-QA-1 — `audit-event-type-emission.test.ts`

**Reproduce**: `pnpm vitest run tests/unit/broadcasts/application/audit-event-type-emission.test.ts`

**Expected**: All 54 declared F7AuditEventType events have ≥1 emit site in `src/`

**Actual** (pre-fix): 6 events declared but never emitted — `broadcast_body_image_source_unsafe`, `broadcast_image_too_large`, `broadcast_image_allowlist_updated`, `broadcast_template_created`, `broadcast_template_updated`, `broadcast_template_deleted`

**Root cause**: F71A Phase 2 (commit `b9af9e9`) added 10 audit events to the `F7_AUDIT_EVENT_TYPES` tuple + migration 0167 in preparation for US1 + US2 + US7. The 4 US1 events have emit sites (Phase 3 + 3F.11.x). The 6 US2 + US7 events do NOT have emit sites because Phase 4 + Phase 5 are not implemented on this branch (deferred to F7.1a-Phase-2 per Staff Review W-1).

**Fix**: Updated `KNOWN_NOT_YET_EMITTED` const in the test to include the 6 events with explicit "F71A US2 Phase 4 — not implemented" + "F71A US7 Phase 5 — not implemented" comments. When Phases 4-5 ship, the entries get removed and emit-site assertions take over.

### F-QA-2 — `broadcast-state-machine.test.ts` (2 cases)

**Reproduce**: `pnpm vitest run tests/unit/broadcasts/domain/broadcast-state-machine.test.ts`

**Expected**: `BROADCAST_STATUSES` has 8 lifecycle states

**Actual** (pre-fix): 10 lifecycle states — F71A US1 added `partially_sent` + `partial_delivery_accepted` via migration 0169 + 0173

**Root cause**: Domain enum was correctly widened from 8 → 10 in F71A Phase 2 + 3 commits. Unit test assertion was not updated alongside the enum widening — test-expectation drift.

**Fix**: Updated `BROADCAST_STATUSES.toEqual(...)` to include all 10 states + `toHaveLength(10)`. Added inline comment that state-machine correctness for the 2 new states is covered by `cancel-cutoff-policy.test.ts` + `broadcast-phase.test.ts`.

**Both failures are TEST-EXPECTATION DRIFT, NOT LOGIC REGRESSIONS.** Production code is correct; tests lagged the F71A US1 enum changes.
