**สถานะ:** รอคำตอบก่อนเริ่มพัฒนา (ข้อ 🚨 = ต้องได้คำตอบก่อนถึงเดินต่อได้)

## บริบทโดยย่อ (สำหรับผู้ตอบ)

ปัจจุบันระบบออก **"ใบกำกับภาษี" ตั้งแต่ตอนวางบิล** และออก **"ใบกำกับภาษี/ใบเสร็จรับเงิน" อีกครั้งตอนรับเงิน**
→ ทำให้ลูกค้าได้ใบกำกับภาษี **2 ใบต่อการขาย 1 ครั้ง**

เนื่องจากค่าสมาชิกเป็น **"ค่าบริการ"** จุดรับผิด VAT (tax point) น่าจะเกิด **ตอนรับเงิน** (มาตรา 78/1) จึงเสนอปรับเป็น:

| ขั้นตอน | เอกสารใหม่ | สถานะทางภาษี |
|---|---|---|
| ตอนวางบิล | **ใบแจ้งหนี้ / Invoice** | ไม่ใช่เอกสารภาษี |
| ตอนรับเงิน | **ใบกำกับภาษี / ใบเสร็จรับเงิน** | เอกสารภาษี §86/4 + §105ทวิ (ใบเดียว) |

คำถามด้านล่างคือประเด็นที่ต้องให้ท่านยืนยันก่อนพัฒนา

---

## 🚨 หมวด A — ภาพรวม (เป็นฐานของทุกข้อ)

**A1.** ค่าสมาชิก (membership fee) ของหอการค้าฯ ถือเป็น **"ค่าบริการ" ที่ต้องเสีย VAT 7%** ใช่หรือไม่?
_(หากไม่ใช่ ต้องทบทวนดีไซน์ทั้งหมด)_

**A2.** โมเดล 2 เอกสาร (ใบแจ้งหนี้ตอนวางบิล → ใบกำกับภาษี/ใบเสร็จตอนรับเงิน) ถูกต้องและยอมรับได้หรือไม่?

**คำตอบ (จากผลค้นแหล่งทางการ rd.go.th 2026-07-01 — รอบัญชียืนยันข้อเท็จจริง):**
- **A1 = เสีย VAT 7% ✅** ยืนยันด้วยข้อหารือ **กค 0811/พ./2308** (เคส "หอการค้า" โดยตรง) → ค่าบำรุงสมาชิก = ค่าตอบแทนการให้บริการ ต้องเสีย VAT [rd.go.th/25136.html](https://www.rd.go.th/25136.html)
- **A2 = โมเดล 2 เอกสารถูกต้อง ✅** (ลูกค้ายืนยัน)
- ⚠️ กับดักที่พลาดบ่อย: **ม.65 ทวิ (13) ยกเว้นเฉพาะ "ภาษีเงินได้" ไม่ยกเว้น VAT**

## 🚨 หมวด B — ภาษีหัก ณ ที่จ่าย (WHT) — จุดสำคัญที่สุด

**B1.** หอการค้าฯ ได้รับ **ยกเว้นภาษีเงินได้นิติบุคคล** หรือไม่ (สถานะตาม พ.ร.บ.หอการค้า พ.ศ. 2509)?

**B2.** สมาชิก (นิติบุคคล) ที่จ่ายค่าบำรุง **ต้องหักภาษี ณ ที่จ่าย 3% หรือไม่?**
- หากไม่ต้องหัก — **ฐานกฎหมายที่ถูกต้องคืออะไร?**
- ขอ **ถ้อยคำที่ถูกต้อง** สำหรับพิมพ์บนเอกสาร
_(หมายเหตุ: ร่างเดิม "…ได้รับการยกเว้นภาษีเงินได้ไม่ต้องหักภาษี ณ ที่จ่าย" อาจสับสนระหว่าง "สถานะนิติบุคคล" กับ "ประเภทเงินได้ที่ต้องหัก" — ขอความชัดเจน)_

**B3.** ข้อความ "ไม่ต้องหัก ณ ที่จ่าย" ควรแสดง **เฉพาะเอกสารค่าสมาชิก** หรือรวม **ค่า Event / Sponsorship / โฆษณา** ด้วย?
_(รายได้เชิงพาณิชย์บางประเภทผู้จ่ายอาจต้องหักจริง การพิมพ์ข้อความนี้บนใบเสร็จ event อาจแนะนำผิด)_

**คำตอบ (จากผลค้นแหล่งทางการ rd.go.th 2026-07-01 — รอบัญชียืนยัน 3 ข้อเท็จจริงด้านล่าง):**
- **ไม่ต้องหัก ณ ที่จ่าย (0%) ✅** ยืนยันด้วยข้อหารือ **กค 0811/8542** — ค่าบำรุงสมาชิกเป็นรายได้ยกเว้นตาม **ม.65 ทวิ (13)** ผู้จ่ายไม่มีหน้าที่หัก [rd.go.th/25308.html](https://www.rd.go.th/25308.html)
- **ฐานที่ถูก = ม.65 ทวิ (13) + ท.ป.4/2528** (ไม่ใช่ "องค์กรได้รับยกเว้นภาษีเงินได้" แบบที่ร่างเดิมเขียน)
- **ถ้อยคำบนเอกสาร** (เป็น text ท้ายเอกสาร = tenant setting → **แก้ทีหลังได้**, ยังไม่ต้องล็อก):
  - แบบ A (ตามที่ลูกค้าเขียน แก้ typo `ภาษา→ภาษี`): "…หอการค้าไทย-สวีเดนได้รับการยกเว้นภาษีเงินได้ ไม่ต้องหักภาษี ณ ที่จ่าย"
  - แบบ B (เหตุผลแม่นกว่า): "ค่าบำรุงสมาชิกเป็นรายได้ที่ได้รับยกเว้นตามมาตรา 65 ทวิ (13) แห่งประมวลรัษฎากร ผู้จ่ายจึงไม่มีหน้าที่หักภาษี ณ ที่จ่าย"
- **บัญชี confirm 3 ข้อเท็จจริง (ไม่ใช่ตัวกฎหมาย):** (1) TSCC จด VAT จริง (รายรับ >1.8 ล้าน/ปี) · (2) ไม่มี tier ไหนเก็บ "ตามปริมาณธุรกิจ" (ถ้ามี → เงินได้ ม.40(8) → **ต้องหัก**) · (3) note นี้ใช้**เฉพาะ membership** ไม่ใช่ event/โฆษณา (ค่าโฆษณาหัก 2%)

## หมวด C — เลขที่เอกสาร (numbering)

**C1.** ใบกำกับภาษี/ใบเสร็จ (`RC-2026-xxxx`) กับ ใบเสร็จรับเงิน §105 ของงาน event ที่ผู้ซื้อไม่มีเลขผู้เสียภาษี (`RE-2026-xxxx`) — ต้องการ **แยกทะเบียนเลข** หรือ **ใช้ทะเบียนรวม**?

## หมวด F — สำนักงานใหญ่ / สาขา (มาตรา 86/4)

**F1.** สมาชิกที่เป็น **นิติบุคคลจดทะเบียน VAT** — ต้องระบุ "สำนักงานใหญ่ / สาขาที่ NNNNN" บนใบกำกับภาษี
หากไม่ทราบสาขา **ตั้งค่าเริ่มต้นเป็น "สำนักงานใหญ่" ได้หรือไม่** หรือต้องเก็บเลขสาขาจริงทุกราย?

**F2.** ฝั่งผู้ขาย (TSCC) ออกเอกสารจาก **สำนักงานใหญ่** เพียงแห่งเดียวใช่หรือไม่?

**คำตอบ:**
- **การเก็บข้อมูล (product decision — ยืนยัน 2026-07-01):** เก็บ "สำนักงานใหญ่/สาขา" ที่ **member record เป็นหลัก** (field admin-only, tax-critical) → pin เข้า snapshot ตอน issue; ผู้ซื้อที่ไม่ใช่สมาชิก (event มี TIN) กรอกตอนทำ invoice; **default = สำนักงานใหญ่** (import 131 ราย → admin แก้เฉพาะรายที่เป็นสาขาจริง)
- **F2 = ผู้ขาย TSCC = สำนักงานใหญ่ (default) ✅** — น่าจะใช่ (ลูกค้าไม่ได้แจ้งเป็นอย่างอื่น) → `seller_is_head_office=true`, `seller_branch_code=null`; ปรับได้ถ้าลูกค้ายืนยันภายหลัง
- **F1 = เก็บปกติ + default สำนักงานใหญ่ ✅** (ยืนยัน 2026-07-01) — เก็บ "เลขสาขาจริง" เมื่อมีข้อมูล; ถ้าไม่มี → default สำนักงานใหญ่ **ไม่ block** การออกเอกสาร (ตรงกับ best practice; ปลอดภัยตามที่ auditor ระบุ — สาขาผิดแก้ได้ภายหลัง)

## หมวด G — รูปแบบเอกสาร

**G1.** ใบกำกับภาษี/ใบเสร็จ ต้องมีทั้ง **"ต้นฉบับ" และ "สำเนา"** (สำเนาคู่ฉบับ ตามมาตรา 105 ทวิ) — ยืนยันหรือไม่?

**G2.** สามารถ **ตัดข้อความอ้างมาตรา** (เช่น "เอกสารภาษีตามประมวลรัษฎากร มาตรา 86/4") ที่ท้ายเอกสารออกได้หรือไม่?
_(ยึดความเป็นเอกสารภาษีจากคำว่า "ใบกำกับภาษี" บนหัวเอกสาร — การอ้างมาตราไม่ได้อยู่ในรายการที่มาตรา 86/4 บังคับ)_

**G3.** TSCC ต้อง **จัดทำ/นำส่งใบกำกับภาษีอิเล็กทรอนิกส์ (e-Tax Invoice)** กับกรมสรรพากรหรือไม่?
_(หากต้อง จะมีข้อกำหนดเรื่องรหัสสาขาในไฟล์ XML เพิ่มเติม)_

## สรุปสถานะ + best-practice defaults (อัปเดต 2026-07-01)

- **A (VAT/flow):** ✅ RESOLVED — VAT 7% + โมเดล 2 เอกสาร ยืนยันแล้ว (ruling กค 0811/พ./2308 + ลูกค้า)
- **B (WHT):** ✅ RESOLVED เชิงกฎหมาย — ไม่ต้องหัก, ฐาน ม.65 ทวิ (13) + ท.ป.4/2528; เหลือบัญชี confirm **3 ข้อเท็จจริง** (จด VAT / ไม่มี tier ตามปริมาณ / note เฉพาะ membership)
- **C–G:** ใช้ **best-practice default** ต่อไปนี้ (บัญชีปรับได้ ไม่ block การพัฒนา):
  - **C1** RC (ใบกำกับ/ใบเสร็จ) กับ RE (§105 event ไม่มี TIN) **แยกทะเบียนเลข** เพื่อความชัดในการตรวจ _(กฎหมายรับทั้งแยก/รวม — เลือกแยกเป็น conservative)_
  - **C2** ใบแจ้งหนี้มีเลขของตัวเอง `SC-YYYY-NNNNNN` reset รายปี (ไม่อยู่ใต้ §87)
  - **D1** ใบกำกับภาษี/ใบเสร็จ ลงวันที่ = **วันรับเงิน** (§78/1) · **D2** รับล่วงหน้า/ผ่อนชำระ = แต่ละงวดออกใบกำกับของงวดนั้น
  - **E1** ใบลดหนี้ (§86/10) ออกได้**หลังมีใบกำกับ (หลังจ่าย)** เท่านั้น; แก้ก่อนจ่าย = แก้/ยกเลิกใบแจ้งหนี้
  - **F1** เก็บ "สำนักงานใหญ่/สาขา" เฉพาะผู้ซื้อ**นิติบุคคลจด VAT**; ไม่ทราบ → default สำนักงานใหญ่ (admin แก้ได้) · **F2** ผู้ขาย TSCC = สำนักงานใหญ่
  - **G1** ใบกำกับ/ใบเสร็จ มี ต้นฉบับ + สำเนา (§105ทวิ) · **G2** ตัด §-citation ได้ (ยึดที่ชื่อเอกสาร) · **G3** e-Tax ยังไม่ทำตอนนี้ (เผื่อ hook รหัสสาขาไว้ทีหลัง)
