# คู่มือ: เข้าสู่ระบบ / สิทธิ์ผู้ใช้ (F1 — Auth & RBAC)

> **ใครใช้:** เจ้าหน้าที่ **admin** (สร้าง/ปิด/เปิดบัญชีได้) · **manager** (อ่านอย่างเดียวทุกหน้า เปลี่ยนข้อมูลไม่ได้) · **member** (เข้า portal ดูข้อมูลตัวเอง)
> **เมนู / หน้า:** Staff sign-in (`/admin/sign-in`) · Member sign-in (`/portal/sign-in`) · Users (`/admin/users`) · Account settings (`/admin/account` · `/portal/account`) · Forgot password (`/forgot-password`)
> **UAT ของ flow นี้:** [../../uat/admin/auth-rbac.uat.md](../../uat/admin/auth-rbac.uat.md)

---

## ⚠️ ก่อนเริ่ม — สิ่งที่ต้องมี

1. **ต้องมีบัญชี admin อย่างน้อย 1 บัญชีก่อน** — บัญชี admin แรกสร้างด้วยสคริปต์ bootstrap (ทำครั้งเดียวตอนติดตั้งระบบ ไม่ผ่านหน้าจอ) จากนั้น admin คนแรกจึงเชิญคนอื่นได้
2. **อีเมลต้องส่งออกได้** — flow เชิญผู้ใช้ + ลืมรหัสผ่าน อาศัยการส่งอีเมลจริง (ผ่าน Resend) ถ้าอีเมลไม่ออก ผู้ถูกเชิญ/ผู้ขอรีเซ็ตจะตั้งรหัสไม่ได้
3. **ภาษาแอป:** คู่มือนี้อ้างชื่อปุ่มตามภาษา **อังกฤษ** (ค่า default) ถ้าเจ้าหน้าที่ตั้งแอปเป็นภาษาไทย ปุ่มจะแสดงเป็นไทย — function เดียวกัน
4. **1 บัญชี = 1 บทบาท + อีเมลห้ามซ้ำทั้งระบบ** — คนที่มี 2 บทบาท (เช่น กรรมการที่เป็น CEO บริษัทสมาชิกด้วย) ต้องใช้ **2 อีเมลแยกกัน 2 บัญชี**

---

## ภาพรวม 3 บทบาท (จำให้แม่น)

| บทบาท | เข้า portal ไหน | ทำอะไรได้ |
|---|---|---|
| **admin** | Staff (`/admin`) | เต็มสิทธิ์ — สร้าง/แก้/ลบ/เปลี่ยนสถานะ ทุกโมดูล + จัดการบัญชีผู้ใช้ |
| **manager** | Staff (`/admin`) | **อ่านอย่างเดียวทุกหน้า** — ดู/ดาวน์โหลดได้ แก้ไม่ได้ทุกอย่าง ยกเว้นเปลี่ยนรหัสตัวเอง / ออกจากระบบ / ดูโปรไฟล์ตัวเอง |
| **member** | Member (`/portal`) | ดูข้อมูลบริษัทตัวเองเท่านั้น เข้า staff portal ไม่ได้ |

```
สถานะบัญชี (account status):
[pending] ──ผู้ถูกเชิญใช้ลิงก์ตั้งรหัส──► [active] ──admin กด Disable──► [disabled]
ถูกเชิญแล้ว    (กดยืนยัน Activate account)   ใช้งานได้     (เซสชันถูกตัดทันที)      ปิดใช้งาน
ยังเข้าไม่ได้                                              ▲                          │
                                                          └────admin กด Enable───────┘
                                                               (กลับมาเข้าได้)
```

> 🔴 **กฎเหล็กความปลอดภัย:** ระบบ **ไม่บอก** ว่าอีเมลไหนมีในระบบหรือไม่ — ทั้งหน้า Sign in (รหัสผิด/อีเมลไม่มี/ผิด portal ขึ้นข้อความเดียวกันหมด) และหน้า Forgot password (ขึ้น "ถ้าอีเมลตรงกับบัญชี ลิงก์กำลังส่งไป" เสมอ) นี่คือการป้องกันการเดาอีเมล (anti-enumeration) ตั้งใจให้เป็นแบบนี้

## งานที่ 1 — เข้าสู่ระบบ (เจ้าหน้าที่ admin / manager)

**สถานการณ์:** เจ้าหน้าที่เข้าหน้า staff portal เพื่อทำงานประจำวัน

1. เปิด **`/admin/sign-in`** (Staff portal)
2. กรอก **Email** และ **Password** (เคอร์เซอร์โฟกัสที่ช่อง Email ให้อัตโนมัติ — พิมพ์ได้เลย)
3. กด **"Sign in"** (หรือกด Enter) → ปุ่มจะขึ้นสถานะ **"Signing in…"**
4. เข้าสำเร็จ → ไปหน้าแรกของ staff portal เห็น **ชื่อ + ป้ายบทบาท (role badge)** ของตัวเองที่เมนูมุมขวาบน

> ℹ️ **member** เข้าที่ **`/portal/sign-in`** ขั้นตอนเดียวกัน เข้าแล้วไปหน้า member portal
> ℹ️ ถ้ามีการเปิดหน้าที่ต้องล็อกอินทั้งที่ยังไม่ได้เข้าระบบ ระบบจะเด้งมาหน้า Sign in แล้ว **พากลับไปหน้าเดิม** หลังเข้าสำเร็จ

### ถ้าเข้าไม่ได้ — ข้อความที่อาจเจอ

| ข้อความ | แปลว่า | ทำยังไง |
|---|---|---|
| **"Email or password is incorrect."** | รหัสผิด / อีเมลไม่มี / **เข้าผิด portal** (member มาเข้า staff หรือกลับกัน) — ทั้งหมดขึ้นข้อความเดียวกันโดยเจตนา | ตรวจรหัสอีกครั้ง และเช็คว่าใช้ URL portal ถูก (`/admin/sign-in` สำหรับเจ้าหน้าที่ · `/portal/sign-in` สำหรับสมาชิก) |
| **"Your account has been disabled. Contact an administrator."** | บัญชีถูก admin ปิดใช้งาน | ติดต่อ admin ให้กด Enable |
| **"Too many failed attempts. Try again later."** | ใส่รหัสผิดเกิน **5 ครั้งใน 15 นาที** → บัญชีถูกล็อก **15 นาที** | รอ 15 นาที (ปลดล็อกเอง) หรือใช้ Forgot password |
| **"Too many requests. Please wait and try again."** | ยิงคำขอถี่เกินไป (rate limit) | รอสักครู่แล้วลองใหม่ |

## งานที่ 2 — ออกจากระบบ (Sign out)

1. คลิกที่ **เมนูผู้ใช้ (avatar/ชื่อย่อ)** มุมขวาบน
2. กด **"Sign out"**
3. ระบบจะ **จบเซสชัน** และพากลับไปหน้า Sign in ของ portal นั้น

> 🔴 หลังออกจากระบบ การกด **back/forward ของเบราว์เซอร์ จะไม่เห็นข้อมูลที่ป้องกันไว้** — ต้องเข้าสู่ระบบใหม่
> ℹ️ สมาชิกใน Member portal มีปุ่ม **Sign out** ในหน้า Account (`/portal/account` ส่วน Appearance) ด้วย

---

## งานที่ 3 — หมดเวลาอัตโนมัติ (Idle + Absolute timeout)

ระบบตัดเซสชันเองในกรณีเหล่านี้ (ไม่ต้องทำอะไร — เป็นความปลอดภัย):

- **ไม่ขยับ 30 นาที (idle timeout)** → ก่อนครบ ระบบเด้งกล่อง **"Still there?"** พร้อมนับถอยหลัง 1 นาที มี 2 ปุ่ม
  - **"Stay signed in"** — ต่ออายุเซสชัน ทำงานต่อได้เลย
  - **"Sign out now"** — ออกทันที
  - ถ้าไม่กดอะไรจนครบ → ออกอัตโนมัติ + ขึ้น toast แจ้ง "ออกจากระบบเพราะไม่มีการใช้งาน"
- **เปิดเซสชันเกิน 12 ชั่วโมง (absolute lifetime)** → ตัดเสมอ แม้กำลังใช้งานอยู่ ต้องเข้าใหม่

> ℹ️ การหมดเวลาอัตโนมัติ (idle / 12 ชม.) **ไม่ถูกบันทึกใน audit log** โดยเจตนา (เป็นเรื่องปกติ ถ้าเก็บจะรกเกินไป)

## งานที่ 4 — ลืมรหัสผ่าน → รีเซ็ต (Forgot / Reset password)

**ใช้เมื่อ:** ผู้ใช้ (บทบาทไหนก็ได้) จำรหัสไม่ได้ เข้าระบบไม่ได้

1. ที่หน้า Sign in กดลิงก์ **"Forgot your password?"** → ไปหน้า **`/forgot-password`**
2. กรอก **Email** → กด **"Send reset link"**
3. หน้าจอขึ้น **"If that email matches an account, a reset link is on the way."** (ขึ้นข้อความนี้ **เสมอ** ไม่ว่าอีเมลจะมีจริงหรือไม่ — กันการเดาอีเมล)
4. ถ้าไม่ได้รับอีเมลภายใน 1 นาที จะมีปุ่ม **"Didn't get the email? Resend"** (นับถอยหลัง 60 วินาทีก่อนกดได้) + คำแนะนำให้ดูใน spam folder
5. เปิดอีเมล → คลิกลิงก์ → ไปหน้า **Choose a new password** (`/reset-password/[token]`)
6. กรอก **New password** + **Confirm new password** → กด **"Update password"**
7. สำเร็จ → ขึ้น **"Password updated. You can now sign in."** → เข้าระบบด้วยรหัสใหม่ได้เลย

> 🔴 **ลิงก์รีเซ็ตใช้ได้ครั้งเดียว และหมดอายุใน 1 ชั่วโมง** — เปิดลิงก์เก่า/ลิงก์ที่ใช้แล้วจะขึ้น **"This reset link has expired. Request a new one."** พร้อมลิงก์ **"Request a new link"** ให้ขอใหม่
> 🔴 พอรีเซ็ตสำเร็จ **เซสชันที่เปิดอยู่ทั้งหมดของผู้ใช้นั้นถูกตัด** และรหัสเก่าใช้ไม่ได้ทันที
> ℹ️ รหัสใหม่ต้องยาว **อย่างน้อย 12 ตัวอักษร** และ **ห้ามเป็นรหัสที่เคยรั่วในฐานข้อมูลที่รู้จัก** (ระบบเช็คให้) ไม่ผ่านจะบอกว่าติดกฎข้อไหน

## งานที่ 5 — เชิญผู้ใช้ใหม่ (admin เท่านั้น)

**สถานการณ์:** เพิ่มเจ้าหน้าที่ใหม่ (admin/manager) หรือเปิด portal ให้สมาชิก

1. ไปที่ **Users** (`/admin/users`) → กดปุ่ม **"Invite user"** (มุมขวาบน)
2. ในกล่อง **"Invite a new user"** กรอก:
   - **Email address** (โฟกัสให้อัตโนมัติ)
   - **Role** — เลือก:
     - **"Admin — full access"**
     - **"Manager — read-only on finance"**
     - **"Member — self-service portal"**
   - **Link to member company** (ช่องค้นหา) — ใช้ได้เฉพาะเมื่อเลือก Role = **Member** เพื่อผูกบัญชีกับบริษัทสมาชิก (เลือกหรือเว้นว่างก็ได้)
3. กด **"Send invitation"** → ขึ้น toast **"Invitation sent to …"** บัญชีจะอยู่สถานะ **pending**
4. ผู้ถูกเชิญได้อีเมล → คลิกลิงก์ → ไปหน้า **"Finish setting up your account"** (`/invite/[token]`)
   - ช่อง **Email** ถูกล็อก (อ่านอย่างเดียว) · กรอก **Your name** + **Choose a password** + ยืนยันรหัส → กด **"Activate account"**
   - บัญชีเปลี่ยนเป็น **active** เข้าระบบได้ทันที

> 🔴 **ลิงก์เชิญใช้ได้ครั้งเดียว หมดอายุใน 7 วัน** — เปิดลิงก์หมดอายุ/ใช้แล้ว ขึ้น **"This invitation has expired. Ask an administrator to send a new one."** (ไม่มีปุ่มขอเองได้ — admin ต้องเชิญใหม่)
> ℹ️ **admin ไม่เคยเห็นรหัสผ่านของผู้ถูกเชิญ** — ผู้ถูกเชิญตั้งเอง
> ℹ️ เชิญด้วยอีเมลที่มีในระบบแล้วจะถูกปฏิเสธ พร้อมข้อความ เช่น **"That email is already on the system."**

## งานที่ 6 — ปิด / เปิดบัญชีผู้ใช้ (admin เท่านั้น)

**ใช้เมื่อ:** มีคนลาออก (ปิด) หรือรับกลับ (เปิด)

1. ไปที่ **Users** (`/admin/users`) → หาแถวบัญชีนั้น
2. **ปิด:** กด **"Disable"** → กล่องยืนยัน **"Disable this account?"** อธิบายว่าจะตัดสิทธิ์ทันที + เซสชันที่เปิดอยู่ถูกออกทั้งหมด → กดยืนยัน → ขึ้น toast "… disabled." สถานะเป็น **Disabled**
3. **เปิดกลับ:** บัญชีที่ Disabled จะมีปุ่ม **"Enable"** → กล่องยืนยัน **"Re-enable this account?"** → กดยืนยัน → กลับมา **Active** เข้าระบบได้

> 🔴 **ปิดบัญชี = ตัดทุกเซสชันทันที** บัญชีนั้นจะเข้าระบบไม่ได้และเซสชันที่ค้างอยู่ใช้ต่อไม่ได้
> 🔴 **ปิด/ลดสิทธิ์ admin คนสุดท้ายไม่ได้** — ระบบบังคับให้มี admin ที่ active อย่างน้อย 1 คนเสมอ (รวมถึงห้ามปิดตัวเอง — แถวของตัวเองขึ้น "That's you" ไม่มีปุ่ม)
> ℹ️ ทุกการ Disable/Enable ถูกบันทึกใน audit log

---

## งานที่ 7 — เปลี่ยนรหัสผ่านของตัวเอง (ทุกบทบาท)

**ใช้เมื่อ:** ยังจำรหัสเดิมได้และเข้าระบบอยู่ แต่อยากเปลี่ยนรหัส

1. เปิด **Account settings** — เจ้าหน้าที่ที่ **`/admin/account`** · สมาชิกที่ **`/portal/account`** (ส่วน Account)
2. กรอก **Current password** + **New password** + **Confirm new password** (มีแถบวัดความแข็งแรงรหัสให้ดู)
3. กด **"Change password"**
4. สำเร็จ → ขึ้น toast **"Password updated. Other sessions have been signed out."** — **เครื่องที่กำลังใช้อยู่ยังล็อกอินต่อ** แต่ **เซสชันอื่นทั้งหมดถูกตัด**

> ℹ️ ถ้ารหัสปัจจุบันผิด ขึ้น **"Current password is incorrect."** (ไม่มีอะไรเปลี่ยน)
> ℹ️ รหัสใหม่ห้ามเหมือนรหัสเดิม → **"Pick a different password — the new one cannot equal the current one."**
> ℹ️ ใส่รหัสปัจจุบันผิดถี่ ๆ จะถูก rate-limit เหมือนหน้า Sign in

## งานที่ 8 — manager คือสิทธิ์อ่านอย่างเดียว (อ้างอิงเร็ว)

- manager **เข้า staff portal ได้** เห็นหน้ารายการ/รายงาน/แดชบอร์ด + ดาวน์โหลดได้
- ปุ่มแก้ไข/สร้าง/ลบ/เปลี่ยนสถานะ **ถูกซ่อน** สำหรับ manager (เช่นในหน้า Users ปุ่ม Disable/Enable ถูกซ่อนสำหรับ manager ส่วนปุ่ม "Invite user" แสดงแต่ถูกปิด (disabled กดไม่ได้))
- ถ้ามีการพยายามสั่งงานที่เปลี่ยนข้อมูล (เช่นยิง API ตรง) ระบบ **ปฏิเสธ** และ **บันทึกเป็น audit event** (manager-denied write attempt)

> ℹ️ การเปลี่ยน **บทบาท (role)** ของผู้ใช้ ใน F1 ยังไม่มีปุ่มในหน้าจอ — เป็นการทำงานเบื้องหลัง (ผ่าน API/admin tooling) เมื่อบทบาทเปลี่ยน ผู้ใช้คนนั้นจะถูกบังคับเข้าสู่ระบบใหม่ในการกระทำถัดไป เพื่อให้สิทธิ์ใหม่มีผลสะอาด ๆ (ดู UAT TC-AUTH-13)

## ❓ คำถามที่พบบ่อย / ข้อควรรู้

| คำถาม | คำตอบ |
|---|---|
| ลืมว่าเป็น member หรือ staff เข้า portal ไหน? | หน้า **Forgot password** ใช้ร่วมกัน (`/forgot-password`) ไม่ต้องรู้ portal; แต่หน้า **Sign in** แยก — เจ้าหน้าที่ `/admin/sign-in`, สมาชิก `/portal/sign-in` |
| ใส่รหัสผิดถูกล็อก ต้องรอเท่าไร? | **15 นาที** (ปลดล็อกเอง) หรือรีเซ็ตรหัสผ่านเลยก็ได้ |
| ทำไม Forgot password ไม่บอกว่าอีเมลมีจริงไหม? | ตั้งใจ — กันการเดาว่าอีเมลใดมีบัญชี (anti-enumeration) |
| คนเดียวมี 2 บทบาทได้ไหม? | ไม่ได้ — 1 บัญชี = 1 บทบาท ต้องใช้ 2 อีเมลแยกบัญชี |
| admin เห็นรหัสของผู้ถูกเชิญไหม? | ไม่เห็น — ผู้ถูกเชิญตั้งเองผ่านลิงก์ |
| ปิดบัญชีตัวเองได้ไหม? | ไม่ได้ — แถวของตัวเองไม่มีปุ่ม (ขึ้น "That's you") และห้ามเหลือ admin 0 คน |
| เปลี่ยนรหัสแล้ว ทำไมยังล็อกอินอยู่? | เครื่องปัจจุบันอยู่ต่อโดยตั้งใจ — เฉพาะ **เซสชันอื่น** ที่ถูกตัด |

## 🔴 สรุปสิ่งที่ "ย้อนกลับไม่ได้ / ระวังให้มาก"

1. **กด Disable บัญชี** → ตัดทุกเซสชันของคนนั้นทันที (เปิดกลับได้ด้วย Enable แต่ที่ค้างอยู่หลุดหมด)
2. **รีเซ็ต / เปลี่ยนรหัสผ่าน** → เซสชันอื่น/ทั้งหมดถูกตัด และรหัสเก่าใช้ไม่ได้อีก
3. **ลิงก์เชิญ (7 วัน) และลิงก์รีเซ็ต (1 ชม.) ใช้ได้ครั้งเดียว** → หมดอายุ/ใช้แล้วต้องออกใหม่
4. **admin คนสุดท้ายปิด/ลดสิทธิ์ตัวเองไม่ได้** → ต้องเหลือ admin active ≥ 1 เสมอ
5. **audit log เป็น append-only** → แก้/ลบรายการไม่ได้ เก็บอย่างน้อย 5 ปี
