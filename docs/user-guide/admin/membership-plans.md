# คู่มือ: จัดการแพ็กเกจสมาชิก (Membership Plans) (F2)

> **ใครใช้:** เจ้าหน้าที่สิทธิ์ **admin** (manager ดูได้อย่างเดียว สร้าง/แก้/ลบไม่ได้ · member เข้าหน้านี้ไม่ได้)
> **เมนู:** Membership Plans (`/admin/plans`)
> **UAT ของ flow นี้:** [../../uat/admin/membership-plans.uat.md](../../uat/admin/membership-plans.uat.md)

---

## ⚠️ ก่อนเริ่ม — สิ่งที่ต้องมี

1. **ลงชื่อเข้าใช้ด้วยบัญชี admin** ที่ `/admin/sign-in` — ปุ่มสร้าง/แก้/ลบ/clone ทั้งหมดขึ้นเฉพาะ admin
   - **manager** เปิดหน้า Plans ได้ เห็นรายการ + เปิดดูรายละเอียดแพ็กเกจได้ แต่ **ไม่มีปุ่มแก้ไขใดๆ** และถ้าพิมพ์ URL ตรงไปหน้า new/clone/edit จะถูก **ดีดกลับมาหน้า Plans** (ไม่ใช่ 404)
   - **member** เข้าพื้นที่ admin ไม่ได้เลย (ถูกบล็อกที่ชั้น staff shell)
2. **ต้องมีแพ็กเกจปีปัจจุบันอยู่แล้วในระบบ** (SweCham seed 9 แพ็กเกจปี 2026: corporate 6 + partnership 3) — ถ้ายังไม่มีให้ผู้ดูแลระบบรัน seed ก่อน หรือสร้างใหม่จากงานที่ 4
3. **สกุลเงิน + VAT มาจากการตั้งค่าใบกำกับภาษี (F4)** — ค่าเริ่มต้น SweCham คือ THB / VAT 7% / ค่าแรกเข้า 1,000 บาท ตั้งจาก **Settings → Invoicing** (`/admin/settings/invoicing`) ไม่ใช่จากหน้า Plans
   - ℹ️ F2 **ไม่มี** หน้าแก้ "VAT rate / ค่าแรกเข้า" แยกของตัวเอง — เป็น schema + seed (ตั้งครั้งแรกตอนเปิด tenant) เท่านั้น

---

## ภาพรวมสถานะแพ็กเกจ (จำให้แม่น)

```
[สร้างใหม่/Clone] ──► [Inactive] ──Activate──► [Active] ──Deactivate──► [Inactive]
  (ร่าง/ยังไม่เปิด)     ไม่เปิดให้สมัคร         เปิดให้ F3 สมัครได้      ปิดรับสมัครใหม่
                            │  ▲                                            │
                            │  └────────── Activate ─────────────────────┘
                       Delete (soft) ──► [Deleted] ──Restore──► [Inactive]
                       (ซ่อนจากรายการ      เก็บใน DB เพื่อ audit    กลับมาเป็น Inactive
                        เก็บประวัติไว้)     เปิดดูด้วย Show deleted
```

> 🔵 **Active vs Inactive:** Active = เปิดให้ F3 (สมัคร/ต่ออายุ) เลือกได้; Inactive = ซ่อนจากการสมัครใหม่ แต่ยัง **อ่านได้** เสมอเพื่ออ้างอิงประวัติ (ใบแจ้งหนี้เก่ายัง resolve แพ็กเกจได้)
> 🔵 **Soft-delete เท่านั้น:** ลบแล้วยัง **ไม่หายจริง** — เก็บใน DB พร้อม timestamp เพื่อ audit + พิมพ์ใบเก่าซ้ำ; กู้คืนได้ด้วย **Restore**
> 🔴 **ปีเก่าถูกล็อกบางส่วน (partial lock):** แพ็กเกจที่ปี < ปีปัจจุบัน — ชื่อ/คำอธิบาย/ลำดับ/active flag ยังแก้ได้ แต่ **ราคา/เงื่อนไข/สิทธิประโยชน์ล็อก** ต้อง **Clone มาปีปัจจุบันแล้วแก้ที่ตัวสำเนา** (ดูงานที่ 6)

## ภาพรวมแพ็กเกจ SweCham 2026 (ค่าอ้างอิง seed)

| หมวด | แพ็กเกจ | ค่าสมาชิก/ปี (THB) |
|---|---|---|
| Corporate | Premium / Large / Regular / Start-up / (อื่นๆ) | 36,000 / 26,000 / 16,000 / 10,000 / 6,000 / 1,000 |
| Partnership | Diamond / Platinum / Gold | 200,000 / 150,000 / 100,000 |

> หน้า Plans คิด "ยอดรวม + VAT" ให้ดูตอนแสดงผล โดยอิงอัตรา VAT จากการตั้งค่าใบกำกับภาษี (ค่าที่เก็บในแพ็กเกจเป็นยอด **ก่อน VAT**)

## งานที่ 1 — ดูรายการแพ็กเกจ + กรอง + ค้นหา

**สถานการณ์:** ตรวจสอบว่าปีนี้หอการค้าขายแพ็กเกจอะไรบ้าง ราคาเท่าไร เปิด/ปิดสถานะไหน

1. ไปที่เมนู **Membership Plans** (`/admin/plans`) → เห็นตารางคอลัมน์ **Name · Category · Annual fee · Member type · Year · Status**
2. **ค้นหา** ด้วยช่อง **"Search plans…"** (พิมพ์บางส่วนของชื่อแพ็กเกจ)
3. **กรองหมวด** ที่ dropdown **Category** → เลือก **Corporate** / **Partnership** / **All**
4. **กรองเฉพาะที่เปิดอยู่** ด้วยสวิตช์ **"Active only"**
5. **กรองตามปี** ด้วยการใส่ `?year=2027` บน URL หรือผ่าน command palette / clone flow — ถ้าปีนั้นยังไม่มีแพ็กเกจจะขึ้น empty state **"No plans for this year"**
6. คลิกชื่อแพ็กเกจเพื่อเปิด **หน้ารายละเอียด** (อ่านอย่างเดียว) — เห็น benefit matrix แยกกลุ่ม **Brand Visibility / Events / Partnership Benefits**

> ป้ายสถานะ: **Active** (เปิด) · **Inactive** (ปิด) · **Deleted** (ลบแล้ว — ขึ้นเมื่อเปิด Show deleted)
> ถ้าชื่อแพ็กเกจยังไม่ได้แปลครบ TH/SV จะมีป้าย **"Missing {locales}"** เตือน admin (ไม่ขึ้นกับ manager)

---

## งานที่ 2 — Clone แพ็กเกจทั้งปีไปปีใหม่ (วิธีที่เร็วที่สุด)

**สถานการณ์:** ต้นปีหน้าจะเปิดแพ็กเกจปีใหม่ — ก๊อปทั้งชุดจากปีนี้แล้วค่อยปรับราคา/สิทธิ

1. ที่หน้า Plans กดปุ่ม **"Clone year"** (มุมขวาบน) — หรือกด ⌘K/Ctrl+K แล้วเลือก action **"Clone year"**
2. ในหน้า Clone:
   - **Source year** — ปีต้นทาง (ค่าเริ่มต้น = ปีปัจจุบัน)
   - **Target year** — ปีปลายทาง (ค่าเริ่มต้น = ปีปัจจุบัน + 1)
   - **"Activate cloned plans immediately"** — สวิตช์ (ปกติ **ปิดไว้** เพื่อให้ตรวจก่อนเปิด)
3. กดปุ่ม **"Clone {n} plans"** → ยืนยันในกล่อง
4. สำเร็จ: ระบบสร้างแพ็กเกจปีใหม่จำนวนเท่าต้นทาง (โครงสิทธิ + ราคาเหมือนเดิม) **สถานะ Inactive โดยปริยาย** แล้วพาไปหน้า Plans ที่กรองปีปลายทางพร้อมแก้

> 🔵 Clone จะ **ปฏิเสธ** ถ้าปีปลายทาง **มีแพ็กเกจอยู่แล้ว** — ขึ้นข้อความ "Target year {year} already has plans…" (กันสร้างซ้ำเงียบๆ)
> 🔵 Clone ปีที่ **ไม่มีแพ็กเกจเลย** ก็ปฏิเสธ — ขึ้น "Source year {year} has no plans to clone."
> 🔵 แก้ราคาปีใหม่ **ไม่กระทบปีเก่า** — ใบแจ้งหนี้ปี 2026 ยังอ้างราคาเดิมเสมอ

---

## งานที่ 3 — เปิด/ปิดการใช้งานแพ็กเกจ (Activate / Deactivate)

**สถานการณ์:** เปิดแพ็กเกจปีใหม่ให้สมัครได้ หรือปิดแพ็กเกจที่เลิกขาย

1. ที่แถวแพ็กเกจในตาราง กดไอคอน **⋯ (More)** ท้ายแถว
2. - แพ็กเกจ Inactive → เลือก **"Activate"** → ทำงานทันที (ไม่มีกล่องยืนยัน เพราะไม่ใช่การทำลาย)
   - แพ็กเกจ Active → เลือก **"Deactivate"** → ขึ้นกล่องยืนยัน **"Deactivate {plan}?"** → กด **"Deactivate"**
3. สำเร็จ: ขึ้น toast + ป้ายสถานะเปลี่ยน

> 🔵 Deactivate = สมาชิกเดิมที่อยู่บนแพ็กเกจ **ยังใช้ได้** แต่ **ห้ามสมัครใหม่** จนกว่าจะ Activate ใหม่
> 🔵 Activate กดรัวๆ ไม่เกิดการทำงานซ้ำ (ระบบมี in-flight guard (กันดับเบิลคลิก))

---

## งานที่ 4 — สร้างแพ็กเกจใหม่จากศูนย์ (Create wizard)

**ใช้เมื่อ:** เพิ่ม tier ใหม่ที่ไม่มีในปีก่อน (กรณีทั่วไปให้ใช้ Clone งานที่ 2 แทน)

1. ที่หน้า Plans กดปุ่ม **"New plan"** (มุมขวาบน) — หรือ ⌘K → action **"Create new plan"**
2. เดิน wizard ทีละขั้น — แต่ละขั้น validate ฟิลด์ของตัวเองก่อนไปต่อ:
   - **Basics** — Plan ID (slug ตัวพิมพ์เล็ก/เลข/ขีด เช่น `premium`), Plan name (3 ภาษา EN บังคับ / TH / SV), Description, Sort order, Category, Member type
   - **Fees** — Annual fee, (Partnership) Bundled corporate plan, Min/Max turnover, Max duration, Max member age
   - **Benefits** — benefit matrix (e-blast/ปี, cultural tickets, directory listing, discount scope ฯลฯ)
   - **Review** — ตรวจทั้งหมดก่อนบันทึก
3. ปุ่ม **"Next"** / **"Back"** เดินขั้น; ที่ขั้นสุดท้ายกด **"Save plan"**
4. สำเร็จ: ขึ้น toast "Plan "{name}" created." แล้วกลับหน้า Plans

> 🔵 **EN ของชื่อแพ็กเกจบังคับกรอก** — เว้นว่างจะ save ไม่ได้และไฮไลต์แท็บภาษาที่ขาด (TH/SV ปล่อยว่างได้ แต่จะมีป้าย Missing เตือน)
> 🔵 Plan ID ต้อง **ไม่ซ้ำ** ภายใน (tenant + ปี) — ซ้ำจะขึ้น "A plan with this ID already exists for {year}."

---

## งานที่ 5 — แก้ไขแพ็กเกจ (ปีปัจจุบัน)

**สถานการณ์:** แก้ราคา/ชื่อ/สิทธิ ของแพ็กเกจปีนี้

1. ที่แถวแพ็กเกจ กด **⋯ → "Edit"** (หรือเปิดหน้ารายละเอียดแล้วแก้)
2. แก้ฟิลด์ที่ต้องการ → กด **"Save changes"**
3. สำเร็จ: ขึ้น toast "Plan "{name}" updated." + บันทึก audit `plan_updated` พร้อม diff ก่อน/หลัง

> 🔵 ถ้าไม่ได้แก้อะไรเลยแล้วกด Save จะขึ้น "No changes to save." (ระบบส่งเฉพาะฟิลด์ที่เปลี่ยนจริง — audit สะอาด ไม่มี write หลอกๆ)
> ℹ️ สองคนแก้แพ็กเกจเดียวกันพร้อมกัน = **last-write-wins** + toast เตือนคนที่ถูกทับว่าใครทับ

---

## งานที่ 6 — แก้แพ็กเกจปีเก่า (ติดล็อก → Clone เพื่อแก้)

**ใช้เมื่อ:** ต้องแก้ **ราคา/เงื่อนไข/สิทธิ** ของแพ็กเกจปีที่ผ่านมาแล้ว

1. เปิด **Edit** แพ็กเกจปีเก่า → จะเห็น **แถบเตือน (banner)** ว่าแพ็กเกจปีเก่าถูกล็อก
   - แก้ได้: **ชื่อ (ทุกภาษา) · คำอธิบาย · Sort order · active flag · ลบ/กู้คืน**
   - ล็อก (แก้ไม่ได้): **Annual fee · ค่าแรกเข้า · Min/Max turnover · Max duration · Max member age · Member type · benefit matrix · bundled corporate**
2. ถ้าพยายามบันทึกฟิลด์ที่ล็อก ระบบ **ปฏิเสธ** พร้อมข้อความที่ระบุชื่อฟิลด์
3. กดปุ่มในแถบเตือน **"Clone to {currentYear}"** → ก๊อปแพ็กเกจมาปีปัจจุบัน แล้วแก้ที่สำเนาแทน

> 🔴 แพ็กเกจปีเก่าเป็น **source of truth ของใบแจ้งหนี้เก่า** — ห้ามแก้ราคาทับ; สำเนาปีปัจจุบันจะกลายเป็นตัวที่ F3 ใช้สมัครใหม่ ส่วนตัวเก่าคงไว้เพื่อ audit/พิมพ์ซ้ำ

---

## งานที่ 7 — ลบ (soft-delete) + กู้คืน (Restore)

**ใช้เมื่อ:** เลิกใช้แพ็กเกจถาวร แต่ยังต้องเก็บประวัติ

1. **ลบ:** ที่แถว กด **⋯ → "Delete"** → กล่องยืนยัน **"Delete {plan}?"** → กด **"Delete"**
   - แพ็กเกจหายจากรายการ default แต่ยังอยู่ใน DB (มี deletion timestamp) + บันทึก audit
2. **ดูที่ลบแล้ว:** เปิดสวิตช์ **"Show deleted"** (เฉพาะ admin) → แพ็กเกจที่ลบขึ้นมาแบบสีจาง พร้อม action **"Restore"**
3. **กู้คืน:** ที่แถวที่ลบ กด **⋯ → "Restore"** → กล่องยืนยัน **"Restore {plan}?"** → กด **"Restore"** → กลับมาเป็น **Inactive**

> 🔴 ลบแพ็กเกจที่ยังมี **สมาชิก active ผูกอยู่ไม่ได้** (เมื่อ F3 มีข้อมูลสมาชิก) — ระบบปฏิเสธพร้อมบอกจำนวนสมาชิกที่ต้องย้ายก่อน
> 🔵 F2 **ไม่มี hard-delete** — ลบจริงต้องติดต่อผู้ดูแลแพลตฟอร์ม (เพื่อ audit compliance)

---

## งานที่ 8 — Command Palette (⌘K / Ctrl+K)

**สถานการณ์:** กระโดดไปแพ็กเกจ/สั่งงานเร็วๆ โดยไม่ต้องคลิกเมนู

1. กด **⌘K** (Mac) หรือ **Ctrl+K** (Windows/Linux) จากหน้าไหนก็ได้ใน admin → กล่องค้นหาเด้งขึ้น โฟกัสที่ช่องค้นหา
2. พิมพ์ชื่อแพ็กเกจ (ตั้งแต่ 1 ตัวอักษรขึ้นไป) (เช่น `plat`) → ผลขึ้นแยกกลุ่ม **Plans / Actions / Navigate**
3. ใช้ลูกศร **↑/↓** เลื่อน, **Enter** เลือก, **Esc** ปิด (โฟกัสกลับไปจุดเดิม)
4. Actions ของ F2: **"Create new plan"** และ **"Clone year"** — ขึ้นเฉพาะ **admin** (manager ไม่เห็น action เขียนข้อมูล)

> 🔵 manager เปิด palette ได้ แต่จะเห็นแค่ search + navigate (อ่าน) ไม่เห็น action สร้าง/clone
> 🔵 ถ้าเปิด reduced-motion ไว้ แอนิเมชันเปิด/ปิด palette จะถูกลด/ตัดออกตามมาตรฐาน a11y

| คำถาม | คำตอบ |

| เปลี่ยน VAT / ค่าแรกเข้า ที่ไหน? | F2 **ไม่มี** หน้าแก้ของตัวเอง — ตั้งที่ **Settings → Invoicing** (F4); ค่า SweCham = VAT 7% / ค่าแรกเข้า 1,000 |
| เปลี่ยน **สกุลเงิน** ได้ไหม? | **ไม่ได้** เมื่อมีแพ็กเกจอยู่แล้ว (จะตีความเลข minor-unit ผิด ~100 เท่า) — ตั้งครั้งเดียวตอนเปิด tenant |
| manager ทำไมไม่เห็นปุ่มแก้/สร้าง? | manager เป็น **อ่านอย่างเดียว** — เห็นรายการ+รายละเอียดได้ แต่ไม่มี controls; พิมพ์ URL ตรงไปหน้า new/clone/edit จะถูก **ดีดกลับหน้า Plans** |
| member เปิดหน้า Plans ได้ไหม? | ไม่ได้ — ถูกบล็อกที่ชั้น staff (ไม่เปิดเผยว่าแพ็กเกจมีอยู่จริง) |
| ลบแพ็กเกจแล้วหายจริงไหม? | ไม่ — **soft-delete** เก็บใน DB เพื่อ audit; กู้คืนด้วย **Restore** จากสวิตช์ **Show deleted** |

## 🔴 สรุปสิ่งที่ "ต้องระวัง / ย้อนยาก" (ดูให้ชัวร์ก่อน)

1. **Clone ไปปีที่มีแพ็กเกจอยู่แล้ว → ถูกปฏิเสธ** (กันซ้ำ) — เลือกปีปลายทางที่ว่าง
2. **แพ็กเกจปีเก่าล็อกราคา/สิทธิ** → ต้อง Clone มาปีปัจจุบันเพื่อแก้ (อย่าแก้ทับประวัติ)
3. **สกุลเงินเปลี่ยนไม่ได้เมื่อมีแพ็กเกจแล้ว** (data-integrity hazard)
4. **Delete = soft-delete** เท่านั้น (กู้คืนได้); ไม่มี hard-delete ใน F2
5. ลบแพ็กเกจที่มี **สมาชิก active ผูกอยู่ไม่ได้** — ย้ายสมาชิกก่อน
