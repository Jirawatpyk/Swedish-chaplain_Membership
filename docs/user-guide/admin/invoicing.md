# คู่มือ: ออกใบกำกับภาษี / ใบเสร็จ / ใบลดหนี้ (F4)

> **ใครใช้:** เจ้าหน้าที่สิทธิ์ **admin** (manager ดู/ดาวน์โหลดได้อย่างเดียว ออก/แก้ไม่ได้)
> **เมนู:** Invoices (`/admin/invoices`) · Credit Notes (`/admin/credit-notes`)
> **UAT ของ flow นี้:** [../../uat/admin/invoicing.uat.md](../../uat/admin/invoicing.uat.md)

---

## ⚠️ ก่อนเริ่ม — สิ่งที่ต้องมี

1. **ตั้งค่าใบกำกับภาษีของหอการค้าให้ครบก่อน** (ทำครั้งเดียว) ที่ **Settings → Invoicing** (`/admin/settings/invoicing`):
   - ชื่อนิติบุคคล (ไทย + อังกฤษ), เลขประจำตัวผู้เสียภาษี 13 หลัก, ที่อยู่จดทะเบียน (ไทย + อังกฤษ)
   - อัตรา VAT (ปกติ **7%**), รูปแบบเลขใบเสร็จ (รวม/แยก), นโยบาย pro-rate, จำนวนวันครบกำหนด (เช่น 30)
   - หากตั้งค่าไม่ครบ หน้า Invoices จะขึ้นกล่อง **"Configure Invoicing"** และ **ออกใบไม่ได้**
2. ระบบ F4 ต้องเปิดอยู่ (`FEATURE_F4_INVOICING` — ปกติเปิด)

---

## ภาพรวมสถานะใบกำกับภาษี (จำให้แม่น)

```
[Draft] ──Issue…──► [Issued] ──Record payment──► [Paid]
ร่าง      (กินเลขที่รัน)     ออกแล้ว    (ออกใบเสร็จ)        ชำระแล้ว
แก้ได้     (ย้อนไม่ได้!)      ค้างชำระ                        │
ลบได้                          │                              ├─ออกใบลดหนี้เต็ม─► [Credited]
                          Void…                              └─ออกใบลดหนี้บางส่วน─► [Partially credited]
                          (เฉพาะที่ยังไม่ชำระ)
                               ▼
                            [Void] ยกเลิก (สิ้นสุด ย้อนไม่ได้)

หมายเหตุ: "Overdue/เกินกำหนด" เป็นสถานะที่ระบบคำนวณเอง (Issued + เลยวันครบกำหนด) — หายเองเมื่อชำระ/ยกเลิก
```

> 🔴 **กฎเหล็ก:** ทันทีที่กด **Issue** ระบบจะ "กินเลขที่รันใบกำกับภาษี" ตามลำดับ §87 ของกรมสรรพากร — **เลขนี้ย้อนกลับ/นำกลับมาใช้ใหม่ไม่ได้** แม้จะ Void ภายหลัง ก่อนกด Issue ให้ตรวจให้ชัวร์ด้วยปุ่ม **Preview** ก่อน (ดูงานที่ 1 ขั้นที่ 5)

---

## งานที่ 1 — ออกใบกำกับภาษี (สร้างร่าง → Issue)

**สถานการณ์:** ออกใบแจ้งหนี้ค่าสมาชิกเต็มปีให้บริษัทสมาชิก

1. ไปที่ **Invoices** → กดปุ่ม **"New invoice"** (มุมขวาบน)
2. ที่ตัวเลือกแบบ radio (legend **"What is this invoice for?"**) เลือก **Membership** (หรือ **Event fee** สำหรับค่าร่วมงาน)
3. กรอกข้อมูลที่จำเป็น:
   - **Member** — เลือกบริษัทสมาชิก
   - **Plan** — แพ็กเกจ/ปี (เฉพาะใบค่าสมาชิก)
   - **Issue date** (วันที่ออก) และ **Due date** (วันครบกำหนด — ระบบเติมให้ตามวัน default)
   - **Line items** — รายการ: คำอธิบาย (ไทย + อังกฤษ), จำนวน, ราคาต่อหน่วย
     - ระบบเติมรายการ **ค่าสมาชิก** ให้ และ **ค่าแรกเข้า** (ถ้าเป็นสมาชิกใหม่รอบนี้)
4. ดู **ตัวอย่าง VAT 7%** และยอดรวมที่คำนวณสดด้านล่างฟอร์ม → กด **บันทึกเป็นร่าง (Draft)**
5. **ตรวจก่อนออกจริง:** ในหน้ารายละเอียดร่าง กด **"Preview"**
   - ได้ไฟล์ลายน้ำคำว่า **"PREVIEW"** → ตรวจชื่อ/เลขผู้เสียภาษี/รายการ/ยอดให้ครบ
     - หมายเหตุ: spec กำหนดข้อความลายน้ำเป็นอีกแบบ ("DRAFT / ร่าง — NOT A TAX DOCUMENT") — เป็น divergence ที่บันทึกติดตามไว้
   - ✅ Preview **ไม่กินเลขที่รัน ไม่บันทึก** แก้กี่รอบก็ได้
6. เมื่อถูกต้องแล้ว กด **"Issue…"** → ในกล่องยืนยัน **พิมพ์คำว่า "ISSUE"** เพื่อยืนยัน (ร่างยังไม่มีเลขเอกสาร จึงพิมพ์คำว่า ISSUE ไม่ใช่เลขเอกสาร) แล้วกดปุ่ม **"Issue"** (ต้องยืนยัน เพราะย้อนไม่ได้)
   - ระบบจะ: กำหนด **เลขใบกำกับภาษี** (เช่น `INV-2026-000001`) · สร้าง PDF (ไทย/อังกฤษ) · ส่งอีเมลให้สมาชิก (ถ้าเปิด auto-email)
   - สถานะเปลี่ยนเป็น **Issued**

> ถ้ายังไม่อยากออก: กด **"Delete draft"** เพื่อลบร่างทิ้ง (ไม่กินเลข) — นี่เป็นวิธีเดียวที่ลบร่างได้

---

## งานที่ 2 — บันทึกการชำระเงิน → ออกใบเสร็จ

**สถานการณ์:** สมาชิกโอนเงินชำระใบที่ออกไปแล้ว

1. เปิดใบที่สถานะ **Issued** → กด **"Record payment"**
2. กรอก:
   - **วันที่ชำระ** (เช่น 2026-01-20)
   - **วิธีชำระ** — bank transfer / cheque / cash / other
   - **เลขอ้างอิง** (เช่น `SCB-20260120-4421` หรือเลขเช็ค)
   - **หมายเหตุ** (ถ้ามี)
3. ยืนยัน → สถานะเปลี่ยนเป็น **Paid** และระบบ **ออกใบเสร็จ** (กินเลขใบเสร็จถ้าใช้โหมดแยกเลข)
4. ดาวน์โหลดเอกสารจากเมนู **"⋯" (More)** ในหน้ารายละเอียด (รายการ **"Download receipt PDF"** / **"Download invoice/receipt"**) — หน้ารายละเอียด **ไม่มีปุ่ม Download แบบลอยๆ**; ส่วนหน้ารายการ (list) มีลิงก์ดาวน์โหลดราย row คือ **"Invoice"** / **"Receipt"**. ป้ายบนเอกสารจะเป็น **"ใบเสร็จรับเงิน / Official Receipt"** (หรือแบบรวม "ใบกำกับภาษี/ใบเสร็จรับเงิน" ตามโหมดที่ตั้งไว้)

> 🔴 บันทึกชำระแล้ว **"ยกเลิกการชำระ" ไม่ได้** — ถ้าผิดต้องใช้ **ใบลดหนี้ (Credit Note)** เท่านั้น
> ℹ️ กดชำระซ้ำใบเดิมจะถูกปฏิเสธ (ระบบกันออกใบเสร็จซ้ำ)

---

## งานที่ 3 — ออกใบลดหนี้ (Credit Note) สำหรับใบที่ชำระแล้ว

**ใช้เมื่อ:** ต้องคืนเงิน/ลดหนี้ใบที่ **ชำระแล้ว** (เช่น สมาชิกยกเลิกกลางปี)

1. เปิดใบสถานะ **Paid** → กด **"Issue credit note"**
2. กรอก:
   - **จำนวนเงินที่ลด** (ไม่เกินยอดคงเหลือที่ยังลดได้) — VAT จะคำนวณตามสัดส่วนให้อัตโนมัติ
   - **เหตุผล** (บังคับ)
3. **พิมพ์คำว่า "CREDIT"** ในกล่องยืนยัน (ปุ่มยืนยันจะ enable เมื่อพิมพ์ตรงเท่านั้น — `admin.creditNotes.new.confirmPhrase = "CREDIT"`) แล้วกดปุ่ม **"Issue credit note"** → ระบบสร้าง **ใบลดหนี้** (เลขที่รันของตัวเอง เช่น `CN-2026-000001`, ป้าย **"ใบลดหนี้ / Credit Note"**)
   - ลด**เต็มจำนวน** → ใบเดิมเป็น **Credited**
   - ลด**บางส่วน** → ใบเดิมเป็น **Partially credited** (ออกใบลดหนี้เพิ่มได้จนเต็ม)
4. ดูใบลดหนี้ทั้งหมดได้ที่เมนู **Credit Notes** หรือในส่วน "Credit notes" ของหน้าใบเดิม

> ใบลดหนี้ **แก้ไม่ได้หลังออก** — ถ้าจะปรับเพิ่มให้ออกใบลดหนี้ใบใหม่กับยอดคงเหลือ
> ออกใบลดหนี้กับใบที่ **ยังไม่ชำระ** ไม่ได้ → ให้ใช้ **Void** แทน (งานที่ 4)

---

## งานที่ 4 — ยกเลิกใบกำกับภาษี (Void)

**ใช้เมื่อ:** ใบที่ **ออกแล้วแต่ยังไม่ชำระ** ออกผิด/ไม่ต้องการแล้ว

1. เปิดใบสถานะ **Issued** → กด **"Void…"**
2. ในกล่องยืนยัน: **พิมพ์เลขเอกสาร** ของใบนั้นเพื่อยืนยัน + ใส่ **เหตุผล** (บังคับ)
3. กด **"Void invoice"** → สถานะเป็น **Void**, PDF ถูกประทับ **"VOID / ยกเลิก"**, ส่งอีเมลแจ้งสมาชิก

> 🔴 Void เป็นสถานะ **สิ้นสุด** — ยกเลิกการ Void ไม่ได้, ทำอะไรกับใบนั้นต่อไม่ได้
> เลขของใบที่ Void **ถูกปลดทิ้ง ไม่นำกลับมาใช้** ใบถัดไปจะได้เลขลำดับใหม่
> ใบที่ **ชำระแล้ว** จะ Void ไม่ได้ → ใช้ **Credit Note** (งานที่ 3)

---

## งานที่ 5 — ดาวน์โหลด / ส่งเอกสารซ้ำ

- **Download** — โหลด PDF ของใบกำกับ/ใบเสร็จ/ใบลดหนี้ (ไฟล์เดิมเป๊ะทุกครั้ง — เก็บถาวร). ในหน้ารายละเอียด ดาวน์โหลดผ่านเมนู **"⋯" (More)** (รายการ **"Download invoice PDF"** / **"Download receipt PDF"** / **"Download invoice/receipt"**) — ไม่มีปุ่ม Download แบบลอยๆ; ส่วนหน้ารายการ (list) มีลิงก์ดาวน์โหลดราย row คือ **"Invoice"** / **"Receipt"**
- **Resend** (เมนูไอคอน) — ส่งอีเมลแนบ PDF ให้สมาชิกอีกครั้ง (ส่งซ้ำได้)
- ถ้าอีเมลตอนออก/ชำระ **ส่งไม่สำเร็จ** จะมีแถบเตือนพร้อมปุ่ม **Resend invoice email** (กรณีอีเมลตอนออกใบล้มเหลว) หรือ **Resend receipt email** (กรณีอีเมลตอนชำระล้มเหลว) ในหน้ารายละเอียด (รายการการเงินถูกบันทึกไปแล้ว ไม่กระทบ)

## ❓ คำถามที่พบบ่อย / ข้อควรรู้

| คำถาม | คำตอบ |
|---|---|
| สมาชิก **ไม่มีเลขผู้เสียภาษี** ออกใบได้ไหม? | ได้ (ใบถูกต้องตาม §86/4 ใช้ชื่อ+ที่อยู่) แต่สมาชิกที่จด VAT จะขอภาษีซื้อคืนไม่ได้ถ้าไม่มีเลขบนเอกสาร |
| วันที่บนเอกสารเป็น พ.ศ. หรือ ค.ศ.? | เก็บเป็น ค.ศ. (สากล) เสมอ; แสดงผลภาษาไทยเป็น **ค.ศ. + (พ.ศ.)** เช่น 2026 (2569); PDF แสดงทั้งคู่ |
| ออกใบผิด แต่ยังไม่ชำระ ทำไง? | **Void** (งานที่ 4) แล้วออกใบใหม่ |
| ออกใบผิด แต่ชำระแล้ว ทำไง? | **Credit Note** (งานที่ 3) — Void ไม่ได้ |
| manager ทำไมไม่เห็นปุ่ม Issue/Pay/Void? | manager เป็นสิทธิ์ **อ่านอย่างเดียว** บนการเงิน — ดู/ดาวน์โหลดได้ แต่แก้ไม่ได้ (by design) |
| ลบใบที่ออกแล้วได้ไหม? | ไม่ได้ — ออกแล้วต้อง Void/Credit Note เท่านั้น (ลบได้เฉพาะ **Draft**) |

---

## 🔴 สรุปสิ่งที่ "ย้อนกลับไม่ได้" (ระวังให้มาก)

1. **กด Issue** → กินเลขใบกำกับภาษีถาวร (Preview ก่อนทุกครั้ง)
2. **Record payment** → ออกใบเสร็จถาวร ยกเลิกการชำระไม่ได้
3. **Issue credit note** → ใบลดหนี้แก้ไม่ได้
4. **Void** → สิ้นสุด ยกเลิกไม่ได้
