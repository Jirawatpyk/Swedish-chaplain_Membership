### Data Quality & Privacy Observations

**PII inventory** (feeds into Constitution Principle I — GDPR & data minimization):
- `members.billing_address`, `members.tax_id`, `members.billing_email`
- `contacts.full_name`, `contacts.email`, `contacts.phone`, `contacts.title`, `contacts.role`
- `event_registrations.attendee_name`, `event_registrations.attendee_email`
- `_audit_log.user`

**Payment-adjacent data** (PCI scope boundary):
- `invoices.amount`, `invoices.total`, `invoices.vat`, `invoices.payment_status`, `invoices.tax_receipt_no`
- `event_registrations.receipt_no`
- **No PAN / CVV found** — the workbook only tracks invoice/receipt metadata, not card data.
  This is good: the current scope is **PCI SAQ-A eligible** if payments are delegated to a
  certified processor (Stripe, Adyen, etc.) — no scope expansion needed.

**Currency quirks in timestamps**:
- Several `created_at` / `updated_at` cells show year `2569` (e.g. `2569-04-01 16:38:53`).
  This is the **Thai Buddhist calendar** (2569 BE = 2026 CE). The webapp MUST store
  timestamps in ISO 8601 (Gregorian/UTC) and only convert for display in `th-TH` locale.

**Data integrity gaps seen in current template**:
- `invoices.event_id` is empty for MB invoices (expected) but there's no NOT NULL
  enforcement. The webapp must validate: `invoice_type="Event" ⇒ event_id required`;
  `invoice_type="MB" ⇒ member_id required`.
- `dup_check` is a reactive alert in Excel — in the webapp, `invoice_no` should be
  **UNIQUE constrained** at the DB level.
- `_audit_log` is cell-level (old → new value per cell). The webapp equivalent must be
  **append-only, row-versioned** per Constitution Principle VIII.

**Renewal tracking** is a **derived pivot view**, not a real entity:
- It holds `member_id + year` columns (`inv_2025, status_2025, paid_date_2025, receipt_2025`,
  repeated for 2026/2027/2028).
- In the webapp, this should be a **query / view**, not a table — derive from `invoices`
  filtered by `invoice_type="MB"` + `member_id` + `invoice_year`.

### Migration Implications

1. **8 entities → 8 database tables** (drop `renewal_tracking` as a table; make it a view).
2. **Controlled vocabularies → enums** (payment_status, member_status, ticket_type, etc.)
3. **Formulas → server-side computed fields or DB-generated columns**
   (VAT calc, total, days_overdue, dup_check, display_name sanitization).
4. **VLOOKUP chains → proper FK constraints** with ON DELETE/UPDATE policies.
5. **Excel audit log → append-only `audit_log` table** at the row-version level, not cell level.

7. **Print_Invoice template → server-side PDF generation** (dual Thai/English).
